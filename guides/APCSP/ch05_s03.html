AP CSP — Topic 5.3: Iterating Over Lists

Core Question

How do you turn a raw list of values into a meaningful answer — the average score, the highest grade, whether a target appears, or a filtered subset? The answer is iteration: visit each element and accumulate a result. Knowing which form of iteration to choose, and which of the four fundamental aggregate patterns the problem fits, is one of the most important algorithmic skills you will apply on the AP CSP exam and in the Create Performance Task.

Key Concepts

  • Iterate by element (Form 1): for item in lst: receives each element's VALUE. Preferred when you only need values; matches AP Pseudocode's FOR EACH item IN lst:.
  • Iterate by index (Form 2): for i in range(len(lst)): generates indices 0 through len(lst) - 1. Necessary when position matters (compare neighbors, distinguish first vs. last, update by index).
  • Count pattern: initialize counter = 0; increment when filter holds.
  • Sum / mean pattern: initialize total = 0; add each element; mean = total / len(lst).
  • Maximum / minimum pattern: initialize best = lst[0]; replace when a better element is found. Symmetric for min.
  • Contains-check pattern: initialize found = False; set True when target is matched.
  • Three-part structure: Every aggregate has the same shape — initialize before the loop, update inside, read after.
  • List comprehension (Python only): [expr for x in lst if cond] is a one-line shorthand for the filter-and-collect pattern. AP Pseudocode has no equivalent; exam stimuli use explicit loops.
  • Tuple unpacking: for record in records: date, subject, minutes = record. Naming the fields documents the loop body.
  • len(lst) instead of literals: Hardcoded sizes silently break when the list changes. len() always reports the current length and lets one formula scale.

Worked Examples / Code Walkthroughs

Iterate by element vs. iterate by index

scores = [82, 95, 78, 65, 90, 88, 71, 100, 55, 92]

# Form 1 - by element (values only)
total = 0
for score in scores:
    total = total + score
print(total)              # 816

# Form 2 - by index (need position)
for i in range(len(scores)):
    if i == 0:
        print("First score:", scores[i])
    if i == len(scores) - 1:
        print("Last score:", scores[i])

Use Form 1 when you only need values; switch to Form 2 only when the loop body asks 'where am I?' in the list.

The four aggregate patterns

scores = [82, 95, 78, 65, 90, 88, 71, 100, 55, 92]

# Count: how many fail
failing_count = 0
for score in scores:
    if score < 70:
        failing_count = failing_count + 1
# failing_count == 2

# Sum and mean
total = 0
for score in scores:
    total = total + score
mean = total / len(scores)    # 81.6

# Maximum
best = scores[0]
for score in scores:
    if score > best:
        best = score
# best == 100

# Contains
found = False
for score in scores:
    if score == 100:
        found = True
# found == True

All four follow the same three-part structure: initialize before, update inside, read after. The differences are in what you initialize to and what update rule you apply.

Study Streak Tracker — filter by subject

study_records = []
study_records.append(("2025-09-01", "math", 45))
study_records.append(("2025-09-01", "history", 30))
study_records.append(("2025-09-02", "math", 60))
study_records.append(("2025-09-02", "history", 25))
study_records.append(("2025-09-03", "math", 35))

print("Math sessions only:")
for record in study_records:
    if record[1] == "math":      # record[1] is the subject field (zero-indexed tuple)
        print(record[0], record[2])

# Output:
# 2025-09-01 45 min
# 2025-09-02 60 min
# 2025-09-03 35 min

This is the filter-and-display pattern applied to nested data. record[1] retrieves the second field (subject) of each tuple — Python tuples are zero-indexed just like lists. Tuple indexing inside an outer list loop is the foundation of working with structured records.

List comprehension vs. explicit loop

scores = [82, 95, 78, 65, 90, 88, 71, 100, 55, 92]

# Explicit loop form
passing = []
for score in scores:
    if score >= 70:
        passing.append(score)

# Equivalent list comprehension
passing = [score for score in scores if score >= 70]

# Both produce: [82, 95, 78, 90, 88, 71, 100, 92]

List comprehensions are Python-only. AP exam algorithm traces will only use the explicit loop form, so practice translating between the two.

Common Pitfalls

  • Off-by-one in min/max initialization: Initializing best = 0 when all values could be negative produces the wrong maximum. Use best = lst[0] (the first element) as a safe starting point.
  • Hardcoding the list size: Writing mean = total / 10 instead of total / len(scores) silently breaks when the list grows or shrinks. Always derive the divisor from len().
  • Empty-list division: total / len(lst) raises ZeroDivisionError when the list is empty. Add a guard clause: if len(lst) == 0:.
  • Forgetting that Pseudocode is one-indexed in by-index loops: The Pseudocode equivalent of for i in range(len(scores)) is FOR EACH i IN [1, LENGTH(scores)], not [0, LENGTH(scores) - 1]. See Section 5.2 for the full rule.
  • Using a list comprehension on an exam algorithm-tracing item: The AP exam will only test the explicit loop form. Recognize comprehensions in your own code, but expand them when reasoning about traces.

Connect to the Exam

AP CSP exam questions on iteration test all four aggregate patterns plus the choice between by-element and by-index iteration. Multiple-choice items routinely show a Pseudocode loop and ask what the body computes (count? sum? max? contains?), or show a Python loop and ask for its Pseudocode equivalent. The Create PT awards a rubric point for an Algorithm — typically a count-or-sum pattern over a list — and you must be able to describe its logic in plain English. The Study Streak Tracker's math-filter loop is a textbook filter-and-display pattern; the same shape applies to almost every list-processing question on the exam.
