AP CSP — Topic 1.2: Number Systems and Conversions

Core Question

Why do programmers, hardware engineers, and AP CSP students need to be fluent in three different number systems — decimal, binary, and hexadecimal — when one would seem to suffice? Because each system serves a different purpose: decimal is intuitive for humans; binary maps onto the two-state switches at the heart of every computer; hexadecimal compresses long binary strings into compact, readable notation. Section 1.1 introduced the bit as the universal unit of information; this section gives you the tools to manipulate any quantity expressed in bits, convert fluently between bases, and recognize how computers represent both positive and negative integers. These conversions show up across the rest of the course: in RGB color codes (1.3), overflow boundaries (1.4), and IP addresses (Unit 7).

Key Concepts

  • Base (radix): The number of distinct digit symbols a positional number system uses. Decimal (base 10) uses 0-9; binary (base 2) uses 0-1; hexadecimal (base 16) uses 0-9 and A-F.
  • Place value: Each digit position represents a power of the base, increasing right to left. The number 357 in base 10 means 3100 + 510 + 7*1.
  • Binary (base 2): The native language of digital hardware. Position weights are 1, 2, 4, 8, 16, 32, 64, 128, ... (powers of 2). Example: 1011_2 = 11_10.
  • Hexadecimal (base 16): A shorthand for binary. Each hex digit equals exactly four bits because 16 = 2^4. Conversion is a grouping operation, not arithmetic.
  • Nibble: A 4-bit group, half a byte. The unit that makes binary-to-hex conversion simple: each nibble is one hex digit.
  • Decimal-to-binary conversion: Repeatedly divide by 2 and record remainders; read them bottom-to-top.
  • Python conversions: bin(n) returns '0b...', hex(n) returns '0x...', int(s, base) parses strings in any base. Both int('2A', 16) and int('2a', 16) are valid.
  • Unsigned integer range: An n-bit unsigned integer ranges from 0 to 2^n - 1. 8 bits: 0 to 255.
  • Two's complement (signed integers): The dominant encoding for negative integers. The most-significant bit has weight -2^(n-1). One adder circuit handles both addition and subtraction. 8-bit range: -128 to +127.
  • AP Pseudocode and number bases: The exam writes binary numbers as labeled bit strings (e.g., "the binary string 1010") and supplies decimal equivalents when needed; there is no 0b or 0x literal syntax.

Worked Examples / Code Walkthroughs

Example 1 — Decimal 200 to binary by repeated division:

200 / 2 = 100, remainder 0
100 / 2 = 50,  remainder 0
50  / 2 = 25,  remainder 0
25  / 2 = 12,  remainder 1
12  / 2 = 6,   remainder 0
6   / 2 = 3,   remainder 0
3   / 2 = 1,   remainder 1
1   / 2 = 0,   remainder 1

Reading remainders bottom-to-top: 11001000_2. Verify: 128 + 64 + 8 = 200. Correct.

Example 2 — Decimal 200 to hexadecimal:

Group the binary into nibbles: 1100 1000. Look up each nibble: 1100 = 12 = C; 1000 = 8 = 8. So 200_10 = C8_16. Verify: 12*16 + 8 = 192 + 8 = 200. Correct.

Example 3 — CSS color #FF6600 (orange) to RGB decimal:

r = int('FF', 16)  # 15*16 + 15 = 255
g = int('66', 16)  # 6*16 + 6  = 102
b = int('00', 16)  # 0
print(r, g, b)  # 255 102 0

So #FF6600 means full red, partial green, no blue. The same hex pair format #RRGGBB yields any of 16,777,216 = 256^3 colors.

Example 4 — Binary string '10110011' to decimal:

n = int('10110011', 2)
print(n)  # 179

Verify by hand: 128 + 32 + 16 + 2 + 1 = 179. The int(s, base) function does the work for us in one line.

Example 5 — 8-bit two's complement reading:

For the bit pattern 1111 1111:

  • The leftmost bit has weight -2^7 = -128.
  • The remaining seven bits sum to 64 + 32 + 16 + 8 + 4 + 2 + 1 = 127.
  • Total: -128 + 127 = -1.

In contrast, the same bit pattern interpreted as unsigned would be 255. The interpretation depends on which encoding the program chose.

Common Pitfalls

  • Reading hex digits as decimal. "FF" in hex equals 255 in decimal — not 15, not 99. Each hex digit's value combines with place value: F is 15, but FF is 15*16 + 15 = 255.
  • Wrong direction in repeated-division remainders. Always read remainders BOTTOM-TO-TOP (last to first), not top-to-bottom. Reversing the order gives a wrong binary number with reversed bits.
  • Confusing 4 bits with 4 hex digits. One hex digit = 4 bits, so 4 hex digits = 16 bits = two bytes. A 24-bit color uses 6 hex digits, not 4 or 8.
  • Forgetting the asymmetry of two's complement. 8-bit signed range is -128 to +127, not -127 to +128. There is one more negative value than positive because zero counts as part of the non-negative side.
  • Mixing AP Pseudocode and Python literal syntax. AP Pseudocode does not define 0b or 0x literals; the exam may show a labeled bit string like "00001010" but will not embed a Python literal in pseudocode.

Connect to the Exam

Topic 1.2 supports Big Idea 2 (Data) and feeds directly into the AP exam's algorithmic-tracing problems and any question involving RGB colors, IP addresses, or bitmask logic. You will be asked to convert between bases by hand for small numbers, predict the output of bin(), hex(), or int(s, base) calls, and reason about which range an integer fits in. The exam will not require you to memorize a hex multiplication table, but you should be able to convert a 4-bit nibble to its hex digit instantly. Stimulus-based questions often present a binary or hex literal embedded in code or pseudocode, then ask you to reason about its decimal value or about the consequences of overflow (Section 1.4).

Self-Check

  • Convert decimal 87 to binary using repeated division. Then convert your binary result to hexadecimal by grouping into nibbles. Verify by computing the decimal value of your hex answer.
  • A 16-bit unsigned integer holds the bit pattern 1111 1111 1111 1111. What is the maximum value, and what is the equivalent two's-complement reading?
  • Write the Python expression that converts the binary string '11010101' to its decimal integer value. What does it return?
  • A graphic designer chooses the color #7FFF00 (chartreuse). What are the decimal RGB values? Which channel is at full intensity?
  • Why is it impossible to write 0b2 as a binary literal in Python? Generalize: which digits are allowed in base-b literals?
