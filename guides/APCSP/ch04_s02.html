AP CSP — Topic 4.2: Indefinite Loops

Core Question

What if you do not know in advance how many times a loop must run? Reading user input until a quit command, retrying a network connection until it succeeds, searching a dataset until you find a match — none of these have a known iteration count. An indefinite loop continues as long as some condition holds (or, in AP Pseudocode, until a condition becomes true). The two formulations are logical inverses of each other, and translating between them is the most-tested skill in this section. Mastering Topic 4.2 means writing correct while loops, recognizing when an exit condition can never be reached (an infinite loop), and inverting the condition cleanly when moving to AP Pseudocode.

Key Concepts

  • Indefinite loop: Repeats a block of code for an unknown number of iterations, continuing or stopping based on a Boolean condition evaluated at the start of each cycle.
  • Python while condition:: The body executes while the condition is True. If the condition is False at the start, the body never runs at all.
  • AP Pseudocode REPEAT UNTIL (condition): The body executes while the condition is FALSE, exiting when the condition first becomes TRUE — the logical inverse of Python's while.
  • The inversion rule: To translate while X to REPEAT UNTIL, negate X (< becomes >=, > becomes <=, == becomes !=, != becomes =, and/or flip).
  • Mandatory components of every while loop: initialization before the loop, mutation inside the body — either is sufficient to break correctness if missing.
  • Infinite loop: A loop whose condition never becomes false (Python) or never becomes true (Pseudocode). Most common cause: forgetting to mutate the loop variable.
  • Sentinel value: A special input value that signals "stop processing" — chosen so it cannot appear in legitimate data.
  • Sentinel-controlled loop pattern: while True: ... if value == sentinel: break ... — the cleanest way to handle exit conditions that naturally belong in the middle of the body.
  • Why if/break must come first: If accumulation runs before the sentinel test, the sentinel itself is processed and corrupts the result.
  • AP Chief Reader note: The REPEAT UNTIL / while inversion is the leading source of student errors on indefinite-loop questions on the AP exam.

Worked Examples / Code Walkthroughs

Example 1 — Basic while loop with mandatory mutation

x = 0           # initialization
while x < 5:    # condition tested at top
    print(x)
    x = x + 1   # mandatory mutation — moves x toward failing condition
# Loop exits when x becomes 5; 5 < 5 is False

Output: 0, 1, 2, 3, 4. If the line x = x + 1 is removed, x stays 0 forever and the loop never exits — the canonical infinite-loop bug.

Example 2 — Python ↔ AP Pseudocode translation

x = 0
while x < 10:
    DISPLAY(x)
    x = x + 1

x <- 0
REPEAT UNTIL (x >= 10)
{
    DISPLAY(x)
    x <- x + 1
}

The condition x < 10 is inverted to x >= 10. Both forms iterate exactly 10 times (x = 0 through 9). Asking "which condition causes the loop to stop?" gives x >= 10, which is the condition you place inside REPEAT UNTIL().

Example 3 — Sentinel-controlled accumulation

total = 0
count = 0

while True:
    value = int(input("Enter a number (-1 to quit): "))
    if value == -1:    # sentinel test FIRST
        break          # exit before any accumulation
    total = total + value
    count = count + 1

if count > 0:
    print(f"Average: {total / count:.2f}")
else:
    print("No numbers entered.")

Two design points:

  1. -1 is a valid sentinel for non-negative scores because it cannot be a legitimate data value. For temperatures (which can be negative), use "done" or another out-of-range value.
  2. The if/break runs BEFORE total is updated, so the sentinel value never contributes to the sum.

Example 4 — Inverting a non-numeric condition

# Python:
while not done:
    do_work()
    done = check_status()

# AP Pseudocode:
REPEAT UNTIL (done = TRUE)
{
    do_work()
    done <- check_status()
}

The Python condition not done becomes done (true) in the REPEAT UNTIL form — the negation of not X is X.

Common Pitfalls

  • Forgetting to invert the condition when translating to REPEAT UNTIL. This is the College Board Chief Reader's #1 reported error. Always negate the condition.
  • Forgetting to mutate the loop variable. Without an x = x + 1 (or similar), the condition stays true forever and the program hangs.
  • Putting the sentinel test after accumulation. If you write total = total + value before if value == sentinel: break, the sentinel value gets added to the total. Always test for the sentinel first.
  • Choosing a sentinel that overlaps real data. -1 is fine for ages but corrupts a list of temperatures. Pick a value (or sentinel string) that cannot legitimately appear in the input.
  • Confusing != with ==. AP Pseudocode uses single = for equality in conditions, while Python uses ==. Pseudocode = is comparison; pseudocode <- is assignment.

Connect to the Exam

Topic 4.2 is heavily tested through translation questions: given a while loop, write the equivalent REPEAT UNTIL, or vice versa. Off-by-one errors here come from getting the inverted condition slightly wrong (using > when you should have used >=, for example). Sentinel-controlled patterns appear on free-response questions where students must read input until a stopping signal. Always verify your translation by tracing two iterations by hand and confirming both versions produce the same output. The Chief Reader has flagged this exact translation as the #1 source of point loss on indefinite-loop questions for several years.

Self-Check

  • Translate while score < 100: score = score + roll() to AP Pseudocode. Identify the negation step explicitly.
  • Identify the bug in this code and explain how to fix it: x = 5; while x > 0: print(x). What is missing?
  • Write a sentinel-controlled while loop that reads positive integers from the user, summing them, and stops when the user enters 0. Where do you place the sentinel test?
  • A student writes REPEAT UNTIL (count > 10) when they wanted to translate while count <= 10. What is the bug, and how should they fix the inversion?
  • Explain in your own words why while True: ... break is sometimes preferred to a single while-condition: loop. When does the "natural exit point in the middle of the body" justify this pattern?
