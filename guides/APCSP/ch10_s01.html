AP CSP — Topic 10.1: Pseudocode Boot Camp

Core Question

Why do students who clearly understand a Python algorithm freeze when asked to render the same algorithm in AP Pseudocode (or vice versa)? The Chief Reader reports for the AP CSP exam identify the same cluster of translation mistakes year after year. The gap is not conceptual — it is fluency. This section is a concentrated drill to convert translation from a conscious effort into reflex, and to give you names for the seven specific error patterns that account for most lost translation points.

Key Concepts

  • Off-by-one from 0/1 indexing: Python lists start at index 0; AP Pseudocode lists start at index 1. Translating element access requires shifting the counter or the bound by one.
  • REPEAT UNTIL inverted condition: Python while loops while a condition is true; AP Pseudocode REPEAT UNTIL loops until the condition becomes true. The conditions are opposites — to translate while x < 10, write REPEAT UNTIL (x >= 10).
  • Assignment vs. equality: Python uses = for assignment and == for equality; AP Pseudocode uses <- for assignment and = for equality. Mixing them produces silent bugs (the line parses but does the wrong thing).
  • Loop boundary inclusivity: range(a, b) in Python yields b - a values (excludes b). REPEAT n TIMES runs exactly n iterations. Converting between them requires an explicit counter when index access is needed.
  • Nested-loop trace order: The inner loop runs to completion for every iteration of the outer loop. Total body executions = outer × inner. Always exhaust the inner counter before stepping the outer.
  • Range start/stop/step semantics: range(start, stop, step) defaults to start = 0, step = 1, and excludes stop. range(2, 10, 2) yields 2, 4, 6, 8 — not 10. Always check whether the last expected value reaches stop.
  • >= symbol direction: In rendered pseudocode the symbol may appear as ≥; in Python you type >=. Always double-check the inequality direction during translation — flipping it inverts the loop's behavior.
  • PROCEDURE definition syntax: PROCEDURE name(params) replaces def name(params):. Calls look identical; the definition keyword and RETURN(value) are the visible changes.
  • List operations: LENGTH(list), list[i] (1-indexed), APPEND(list, val), INSERT(list, i, val), REMOVE(list, i). Python equivalents: len, list[i-1], list.append, list.insert(i-1, val), del list[i-1].
  • Boolean keywords: Python and, or, not (lowercase) ↔ AP Pseudocode AND, OR, NOT (uppercase). Semantics identical; case is the only visible change.

Worked Examples / Code Walkthroughs

Example 1 — Counting Loop (Python → Pseudo)

Python:

for i in range(5):
    print(i)

AP Pseudocode:

i <- 0
REPEAT 5 TIMES
  DISPLAY(i)
  i <- i + 1

Note: explicit counter i is required because REPEAT n TIMES has no built-in index. The counter is initialized before REPEAT and incremented after DISPLAY so the displayed values match Python's 0..4.

Example 2 — REPEAT UNTIL Accumulator (Pseudo → Python)

AP Pseudocode:

total <- 0
i <- 1
REPEAT UNTIL (i > 10)
  total <- total + i
  i <- i + 1
DISPLAY(total)

Python:

total = 0
i = 1
while i <= 10:
    total = total + i
    i = i + 1
print(total)

The UNTIL condition (i > 10) becomes while i <= 10 (the negation). The arrow <- becomes =, and DISPLAY becomes print.

Example 3 — Linear Search (Python → Pseudo)

Python:

def search(data, target):
    for i in range(len(data)):
        if data[i] == target:
            return i
    return -1

AP Pseudocode:

PROCEDURE search(data, target)
  i <- 1
  REPEAT UNTIL (i > LENGTH(data))
    IF (data[i] = target)
      RETURN(i)
    i <- i + 1
  RETURN(-1)

Five shifts at once: def → PROCEDURE, 0-indexed → 1-indexed counter, for/range → REPEAT UNTIL, == → =, return → RETURN(...).

Example 4 — Step Range (Python → Pseudo)

Python range(2, 11, 2) yields 2, 4, 6, 8, 10. To translate, count iterations: (11 − 2) / 2 = 4.5, round down → 5 values.

i <- 2
REPEAT 5 TIMES
  DISPLAY(i)
  i <- i + 2

Common Pitfalls

  • Direct copy of while to REPEAT UNTIL without negation: The single most common translation bug. The conditions are opposites, not equivalents. Always negate.
  • Using range(1, n+1) and then accessing data[i] after translating to AP Pseudo: This works in AP Pseudocode (1-indexed) but causes IndexError in Python because Python is 0-indexed. The translation must shift the counter, not just the loop bound.
  • Forgetting the explicit counter in REPEAT n TIMES: AP Pseudocode does not provide an automatic loop variable. If you need the index, declare and increment it manually.
  • Reading = as assignment in AP Pseudocode: In IF (x = 5), the = is an equality test, not an assignment. Writing the same line in Python (if x = 5:) is a syntax error.
  • Ignoring range's exclusive stop: range(1, 6) is five iterations, not six. The last value is stop − step, not stop.

Connect to the Exam

The AP CSP exam tests pseudocode translation in two ways. Direct translation questions show one notation and ask which of four answer choices is the equivalent in the other notation — most distractors are exactly the seven error vectors above. Trace-the-output questions require you to mentally execute pseudocode and predict its display; one of the wrong answers usually corresponds to a 0/1 indexing slip or an off-by-one. The Chief Reader's annual analysis emphasizes that translation skill is now a critical exam-readiness gate. A 10-second pre-scan for all seven error vectors before writing a single line catches most translation bugs before they happen.

Self-Check

  • Translate to AP Pseudocode: for i in range(1, 11): print(i * i). Identify which two error vectors are most likely to trip a translator on this loop.
  • The pseudocode REPEAT UNTIL (count >= 5) with body count <- count + 1 is translated as while count >= 5: count = count + 1. What error vector is this, and what is the correct Python translation?
  • Without writing code, count: how many times does the inner DISPLAY execute when the outer loop is REPEAT 4 TIMES and the inner is REPEAT 6 TIMES?
  • AP Pseudocode data <- [10, 20, 30] then DISPLAY(data[1]). What value is displayed? What would the equivalent Python data = [10, 20, 30]; print(data[1]) display?
  • Write the AP Pseudocode for: def is_even(n): return n % 2 == 0. Highlight every place where the translation differs from a direct character-by-character copy.
