AP CSP — Topic 3.2: Logical Operators and Truth Tables

Core Question

A single relational comparison answers one yes-or-no question. But real decisions usually combine multiple facts: "the user is over 17 AND has a valid subscription," "the player is a registered user OR a guest." How do we combine multiple Boolean values into a single Boolean output, and how do we systematically verify that two compound expressions mean the same thing? This section introduces the three logical operators (and, or, not), shows how truth tables enumerate every possible input combination, and proves logical equivalences using De Morgan's Laws.

Key Concepts

  • Logical operators (Python): and, or, not (lowercase). Combine Booleans into a single Boolean.
  • Logical operators (Pseudocode): AND, OR, NOT (uppercase). Same semantics; case must match the language.
  • Capitalization trap: Writing AND in Python raises NameError: name 'AND' is not defined. Use lowercase in Python, uppercase in Pseudocode.
  • AND: True only when BOTH operands are True. Mnemonic: "and is greedy — it demands both."
  • OR: True when at least ONE operand is True. False only when both are False. This is inclusive or.
  • NOT: Inverts a single Boolean. True becomes False; False becomes True.
  • Truth table: A grid listing every possible combination of Boolean inputs and the resulting output. With n inputs there are exactly 2^n rows.
  • Short-circuit evaluation: Python evaluates and/or left-to-right and stops as soon as the result is fixed. Used as a guard against errors (None checks, division by zero).
  • De Morgan's First Law: not (A and B) ≡ (not A) or (not B). The operator flips when not enters parentheses.
  • De Morgan's Second Law: not (A or B) ≡ (not A) and (not B). The mirror of the first law.
  • Operator precedence: Python evaluates not first, then and, then or. Use parentheses when in doubt.

Worked Examples / Code Walkthroughs

Example 1 — Combining the three operators

age = 22
has_id = True
banned = False

# Allow if old enough AND has ID AND not banned:
allowed = (age >= 21) and has_id and (not banned)
print(allowed)   # True

# AP Pseudocode equivalent:
allowed <- (age >= 21) AND has_id AND (NOT banned)

The and chain requires every operand to be True. not banned flips False to True, so the chain becomes True and True and True = True.

Example 2 — Building a 2-input truth table for A and (not B)

A B not B A and (not B)
F F T F
F T F F
T F T T
T T F F

Only one row produces True (A True, B False). The truth table makes the expression's full behavior unambiguous.

Example 3 — Short-circuit evaluation as a guard

user = None  # No user object exists

# UNSAFE — would crash with AttributeError:
# if user.is_logged_in():
#     print("Welcome!")

# SAFE — and short-circuits when user is None:
if user is not None and user.is_logged_in():
    print("Welcome!")
else:
    print("Please log in.")  # prints safely

Because user is not None is False, Python never evaluates user.is_logged_in(). The guard pattern relies on left-to-right short-circuiting.

Example 4 — De Morgan's First Law

Original: not (x < 5 and y > 10) Apply De Morgan: flip and to or, negate each operand. Rewrite: (x >= 5) or (y <= 10)

You can verify by truth table: in every one of the 4 rows of (x < 5, y > 10), both expressions agree.

Common Pitfalls

  • Uppercase logical operators in Python: AND, OR, NOT raise NameError. Use lowercase.
  • Confusing inclusive vs exclusive OR: Python's or is INCLUSIVE — True when both operands are True. Exclusive OR (XOR) is a separate operator (^ in Python; covered in Topic 3.4).
  • Forgetting that not has higher precedence than and: not A and B parses as (not A) and B, not as not (A and B). Add parentheses when intent matters.
  • Not realizing short-circuiting can hide bugs: if x and slow_function(): will skip slow_function() whenever x is False. That is the point — but it also means the function's side effects do not happen.
  • Misapplying De Morgan (forgetting to flip the operator): not (A and B) = (not A) or (not B), NOT (not A) and (not B). The operator flips.

Connect to the Exam

Topic 3.2 is heavily tested under Big Idea 3 (Algorithms and Programming). Expect: tracing compound Boolean expressions row-by-row through a truth table; computing the number of rows for n inputs (2^n); identifying which expressions are equivalent via De Morgan's Laws (multi-select format: "Which TWO expressions are equivalent to NOT (A AND B)?"); and reading code that depends on short-circuit evaluation (especially guard patterns). Memorize the AND, OR, and NOT truth tables cold — they recur throughout Topics 3.4 (logic gates) and 3.5 (nested conditionals).

Self-Check

  • Build the truth table for A or (not B) with all 4 rows of (A, B). How many rows produce True?
  • A student writes if (score > 90) AND (attempts < 3): in Python. What error appears, and how do you fix it?
  • Apply De Morgan's First Law to not (age >= 18 and has_id). Write the equivalent expression without the outer not.
  • Trace (False or True) and (not False) step by step. Show each sub-result.
  • Why does (x != 0) and (10 / x > 2) not raise ZeroDivisionError when x = 0?
