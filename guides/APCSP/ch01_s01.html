AP CSP — Topic 1.1: The Language of Computers: Bits, Bytes, and Sampling

Core Question

How does a machine built from two-state switches represent the continuous richness of the physical world — sounds that vary smoothly across thousands of pressure changes per second, light that arrives across an unbroken spectrum of wavelengths, the human voice in all its expressive nuance? The answer is the central engineering achievement of the digital age: every analog signal is reduced to a long sequence of binary choices through sampling and quantization, and every form of human communication is ultimately stored as bits. This section establishes the vocabulary and the math that the rest of the course depends on, including Shannon's information-theoretic foundation and the size calculations behind every audio file, image, and video on a device.

Key Concepts

  • Bit (binary digit): The smallest unit of digital information; holds one of two values, 0 or 1. Maps onto the two stable states of an electronic switch. n bits encode 2^n distinct values, so the number of bits grows logarithmically with the number of distinct states required.
  • Byte: 8 bits grouped together; encodes 2^8 = 256 distinct values. The standard "chunk" size of computer memory and the unit in which storage capacity is denominated (KB, MB, GB).
  • Analog signal: A continuously varying quantity (sound pressure, light intensity, voltage). Smooth and uncountable in detail.
  • Digital signal: A discrete sequence of bits, each representing a sample of the underlying analog signal at a particular instant.
  • Sampling rate: Measurements per second when digitizing an analog signal, measured in Hz or kHz. CD audio = 44.1 kHz; phone audio = 8 kHz. Higher rates capture faster variations.
  • Bit depth: Bits used to represent each sample. Higher bit depth = more distinct amplitude levels = smaller quantization error. CD = 16 bits/sample = 65,536 levels.
  • Nyquist-Shannon sampling theorem: A signal containing frequencies up to f Hz can be faithfully reproduced if sampled at a rate of at least 2f Hz. Human hearing tops out near 20 kHz, justifying 44.1 kHz CD audio.
  • Quantization error: The rounding error from snapping each analog measurement to the nearest discrete amplitude level allowed by the bit depth.
  • Shannon entropy: The average number of bits required to encode a message from a source. The lower bound for lossless compression — a message cannot be compressed below its entropy.
  • SI vs. binary prefixes: 1 KB (SI) = 1,000 bytes; 1 KiB (binary) = 1,024 bytes. Drive manufacturers use SI; many OSes use binary, leading to the apparent shrinkage of advertised vs. observed disk capacity.

Worked Examples / Code Walkthroughs

Example 1 — Counting distinct values in n bits:

for n in [1, 4, 8, 16, 24]:
    print(f"{n} bits -> {2**n:,} distinct values")
# 1 bit -> 2 values
# 4 bits -> 16 values
# 8 bits -> 256 values   (one byte)
# 16 bits -> 65,536 values   (CD audio per sample)
# 24 bits -> 16,777,216 values   (24-bit RGB color)

The growth is exponential: each additional bit doubles the count. This is why an extra 8 bits of color depth (24 vs. 16) is so dramatic — it multiplies the number of representable colors by 256.

Example 2 — One second of CD audio in bits:

Sampling rate:  44,100 samples/sec
Bit depth:      16 bits/sample
Channels:       2 (stereo)

Bits per second = 44,100 x 16 x 2 = 1,411,200 bits/sec ~= 1.4 Mb/s
Bytes per second = 1,411,200 / 8 = 176,400 bytes/sec
1 minute  ~=  10.6 MB
4-minute song ~= 42 MB (uncompressed)

Compression (MP3, AAC, Opus) typically reduces this by a factor of 10x for transparent quality.

Example 3 — Raw size of a 24-megapixel photo:

Pixels:          24,000,000
Bits per pixel:  24 (8 each for R, G, B)

Total bits  = 24,000,000 x 24 = 576,000,000 bits
Total bytes = 576,000,000 / 8 = 72,000,000 bytes ~= 72 MB

JPEG compression reduces this to ~5-15 MB. RAW formats (used in pro photography) sit between, preserving more sensor data while staying smaller than fully uncompressed.

Example 4 — Sampling rate vs. perceived quality:

A 48 kHz, 24-bit stereo recording for 3 minutes: 48,000 x 24 x 2 x 180 = 414,720,000 bits ~= 51.84 MB. Doubling that sampling rate to 96 kHz roughly doubles file size but produces no audible improvement for human listeners — Nyquist already covered the audible band at 48 kHz.

Common Pitfalls

  • Confusing 2*n with 2^n. n bits encode 2^n distinct values, not 2n. With n = 8 the difference is 256 vs. 16 — a sixteenfold blunder.
  • Forgetting the channel multiplier in audio calculations. Stereo audio doubles the bit rate over mono. Always multiply by the number of channels.
  • Mixing megabits and megabytes. 1.4 Mb/s (megabits/sec) is one-eighth of 1.4 MB/s (megabytes/sec). Network speeds are usually in bits; file sizes in bytes.
  • Believing higher sampling rate always sounds better. For human listeners and audio in the audible band, anything above ~44.1 kHz is largely inaudible. Storage doubles; perceived quality does not.
  • Treating SI and binary prefixes as identical. A "1 GB" thumb drive (SI) holds about 7% less than a "1 GiB" memory module's capacity (binary). For exact storage calculations, check which convention the document uses.

Connect to the Exam

Topic 1.1 anchors Big Idea 2 (Data) and is foundational for the entire course. Expect AP exam items that ask you to compute the size of a digital file given sampling rate, bit depth, and duration (or pixels and bits per pixel for images), reason about why one digital representation has more or less fidelity than another, and apply the Nyquist-Shannon insight to explain why a particular sampling rate is or is not sufficient. The exam may also test the relationship 2^n distinct values for n bits in a code-trace problem - given n, predict how many states a bit string can encode. Understanding bits as the universal medium of information sets up every following topic on number systems, media encoding, overflow, and compression.

Self-Check

  • A streaming service offers two audio quality tiers: 128 kb/s and 320 kb/s. Compute how much storage each tier consumes for a 1-hour podcast (in MB). Why might the 320 kb/s tier be a better choice for music but unnecessary for a spoken-word podcast?
  • A scanner samples a document at 8 bits per pixel and 300 dots per inch (DPI). For an 8.5-inch by 11-inch page, how many bits will the scan contain? Convert to MB.
  • Why would sampling music at 100 Hz produce unrecognizable audio? Use the Nyquist-Shannon theorem and the typical frequency range of human hearing to justify your answer.
  • Suppose a sensor outputs a continuous voltage between 0 and 5 V. The ADC quantizes the signal at 8-bit depth. What is the smallest voltage difference the ADC can distinguish? (Hint: 5 V divided by 2^8 levels.)
  • Shannon proved that information content equals the number of binary choices needed to specify a message. If a system has 1,024 equally probable messages, how many bits are required to specify which one was chosen? What if there are 1,000 equally probable messages?
