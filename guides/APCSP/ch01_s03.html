AP CSP — Topic 1.3: Digital Representation of Media

Core Question

Numbers are clean — but the messier categories of human communication are not. Letters from dozens of languages, the 70,000+ Han characters of Chinese, full-color photographs, scanned paintings: these media are not natively numeric. To represent them inside a computer, engineers built two great encoding systems: ASCII and Unicode for characters, and the RGB model for colors. Both systems reduce human-meaningful content to integers, which the techniques from Section 1.2 then encode as bits. This section walks through how that reduction works, why backward compatibility (UTF-8 over ASCII) matters in practice, and why character and color encoding are agreed-upon standards rather than facts of nature.

Key Concepts

  • ASCII: American Standard Code for Information Interchange (1963). 7-bit encoding mapping each of 128 codes (0-127) to a character. Covers 95 printable English characters plus 33 control characters. Stored as a byte with the high bit set to 0.
  • ASCII landmarks: 'A' = 65, 'a' = 97, '0' = 48, ' ' = 32. The difference between uppercase and lowercase letters is exactly 32 (= 2^5), so toggling bit 5 flips case.
  • Unicode: A universal standard assigning a unique code point (U+XXXX in hex) to every character in every writing system. Over 149,000 characters as of Unicode 15.1, covering 161 scripts including Han, Devanagari, Arabic, and emoji.
  • UTF-8: A variable-length encoding of Unicode using 1-4 bytes per code point. ASCII range (0-127) uses 1 byte. Higher code points use 2-4 bytes. UTF-8 is backward-compatible with ASCII (every ASCII document is valid UTF-8) and is the web's dominant encoding.
  • Code points vs. bytes: Python 3 len(s) counts code points; len(s.encode('utf-8')) counts bytes. For ASCII text they are equal; for accented or non-Latin characters, bytes can be 2-4 times larger.
  • Pixel: Smallest addressable unit of a digital image. A 1920 x 1080 display has ~2 million pixels. Each pixel stores one color value.
  • RGB: Additive color model: any color is a 3-tuple of intensities for red, green, and blue light. Standard 24-bit color uses 8 bits per channel = 256 levels per channel = 16,777,216 total colors. Written #RRGGBB in hex.
  • Additive vs. subtractive: RGB is additive (mixing emitted light → brighter, white at full); CMYK is subtractive (mixing pigments → darker, black at full). The two are opposites.
  • Mojibake: Garbled text that appears when a file in one character encoding is decoded using another (e.g., UTF-8 read as Latin-1). Illustrates that encoding standards are agreements, not facts.
  • EXIF metadata: Information attached to image files (camera model, GPS coords, timestamps) carried in the file header. Privacy implications when images are shared publicly.

Worked Examples / Code Walkthroughs

Example 1 — ASCII case conversion via arithmetic:

# Convert 'b' (lowercase) to 'B' (uppercase) by toggling bit 5
letter = 'b'
upper = chr(ord(letter) - 32)  # ord('b') = 98; 98 - 32 = 66; chr(66) = 'B'
print(upper)  # B

# Going the other way:
upper = 'X'
lower = chr(ord(upper) + 32)  # ord('X') = 88; 88 + 32 = 120; chr(120) = 'x'
print(lower)  # x

The 32-difference comes from bit 5 (= 2^5 = 32) being the case-distinguishing bit.

Example 2 — Unicode and code points vs. bytes:

word = 'cafe'
cafe = 'café'  # é is U+00E9

print(len(word), len(cafe))               # 4 4 (both have 4 code points)
print(len(word.encode('utf-8')))          # 4 (4 ASCII bytes)
print(len(cafe.encode('utf-8')))          # 5 (é uses 2 bytes in UTF-8)

# Inspecting the bytes:
print(list(cafe.encode('utf-8')))  # [99, 97, 102, 195, 169]
# 99 = 'c', 97 = 'a', 102 = 'f', 195+169 = é (UTF-8 multi-byte sequence)

Example 3 — CSS color #1E90FF (dodger blue) to RGB decimal:

r = int('1E', 16)   # 1*16 + 14 = 30
g = int('90', 16)   # 9*16 + 0  = 144
b = int('FF', 16)   # 15*16 + 15 = 255
print(r, g, b)  # 30 144 255

So dodger blue is "low red, partial green, full blue" — explaining the deep blue with a slight green cast.

Example 4 — Computing 24-megapixel photo file size from first principles:

Total pixels:   24,000,000
Bits per pixel: 24 (8 each for R, G, B)
Total bits  = 24,000,000 * 24 = 576,000,000 bits
Total bytes = 576,000,000 / 8 = 72,000,000 bytes ≈ 72 MB raw
JPEG compressed size: ~5-12 MB (10x to 15x reduction)

Common Pitfalls

  • Forgetting that UTF-8 is variable-length. A non-ASCII character can be 2, 3, or even 4 bytes. Database column sizes, network packet limits, and len() on strings vs. bytes all depend on this.
  • Confusing code point with byte. '😀' is one code point (U+1F600) but 4 bytes in UTF-8. len(s) counts code points; len(s.encode('utf-8')) counts bytes.
  • Misreading hex colors as decimal. #FF6600 is not "red 256, green 66, blue 0." Each pair of hex digits is a byte: 0xFF = 255, 0x66 = 102, 0x00 = 0.
  • Mixing additive and subtractive intuitions. RGB starts at black (#000000) and adds light to reach white (#FFFFFF). CMYK starts at white (paper) and adds pigment toward black. They are opposites.
  • Sharing photos with EXIF intact. GPS coordinates in EXIF can reveal home address, daily routes, or workplace. Most social platforms strip EXIF on upload, but the raw file retains it; private sharing requires manual stripping.

Connect to the Exam

Topic 1.3 anchors Big Idea 2 (Data) and frequently appears in stimulus-style exam questions. Expect: (1) a code segment using ord()/chr() to test understanding of ASCII codes, (2) a question about the size of a digital image given its dimensions and bit depth, (3) a CSS-color hex-to-decimal conversion, and (4) reasoning about why two systems exchanging Unicode text without an agreed encoding produce mojibake. The big-picture concept — that ASCII, Unicode, and RGB are agreed-upon mappings rather than natural facts — is testable in the form of explanation questions about software interoperability.

Self-Check

  • What does ord('Z') return? Verify your answer using the pattern ord('A') = 65 and the alphabetical order of the letters.
  • A web designer specifies the color #3CB371 (medium sea green) in CSS. What are the decimal red, green, and blue values?
  • Explain in one sentence why len('café') returns 4 but len('café'.encode('utf-8')) returns 5.
  • A 4K monitor has 3840 x 2160 pixels. At 24 bits per pixel, what is the raw size of one frame in MB?
  • Why is the difference between uppercase 'A' and lowercase 'a' exactly 32 in ASCII, and how could you exploit this to write a fast case-conversion function in Python without using .upper() or .lower()?
