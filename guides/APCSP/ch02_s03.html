AP CSP — Topic 2.3: Mathematical Expressions

Core Question

How does Python evaluate arithmetic, and where do its rules surprise students who expect calculator behavior? Python is an extraordinarily capable calculator, but precedence rules and the distinction between / (true division) and // (floor division) trip up students on both code-tracing and multiple-choice questions. Every concept in this section also has an AP Pseudocode equivalent — and the keyword MOD appears frequently in exam algorithm traces. Mastering precedence and the integer-vs-float division distinction is the bedrock for every algorithm in Big Idea 3.

Key Concepts

  • Seven arithmetic operators: +, -, *, / (true division, always float), // (floor division), % (modulo, remainder), ** (exponentiation).
  • Modulo (%): Returns the remainder after integer division. 7 % 3 = 1. Used for even/odd checks (n % 2 == 0), cyclic indexing (index % length), and extracting last digits (year % 100). On the AP exam, the same operation is the keyword MOD.
  • PEMDAS in Python: Parentheses first, then exponentiation (right-to-left), then multiplication/division/floor-division/modulo (left-to-right), then addition/subtraction (left-to-right). Adding parentheses never changes a correct expression's meaning but dramatically improves readability.
  • / returns float in Python 3: Even when both operands are integers and the result is whole, 10 / 2 is 5.0, not 5. To get an integer quotient, use //.
  • Floor division (//) rounds toward negative infinity: -7 // 2 = -4 (not -3), because the floor of -3.5 is -4. The % operator follows the same convention: -7 % 2 = 1. This is a frequent multiple-choice trap on Python items; exam MOD problems use positive operands.
  • ** is right-associative: 2 ** 3 ** 2 evaluates as 2 ** (3 ** 2) = 2 ** 9 = 512, not (2 ** 3) ** 2 = 64. Every other arithmetic operator is left-associative.
  • AP Pseudocode equivalents: assignment <-, exponentiation ^, modulo MOD, division /. Notation differs from Python; the math is identical.
  • Big Idea 3 (Algorithms) focus: this section is the prerequisite for everything in Unit 3. The MOD operator alone shows up in tracing problems for divisibility, parity, and circular structures.

Worked Examples / Code Walkthroughs

1. PEMDAS trace. Why does 2 + 3 * 4 produce 14, not 20?

print(2 + 3 * 4)        # 14
print((2 + 3) * 4)      # 20  (parentheses override precedence)

Step 1: * outranks +, so 3 * 4 = 12 evaluates first. Step 2: 2 + 12 = 14. Adding parentheses around 2 + 3 forces the addition first, giving 20.

2. Multi-operator trace. What does 10 - 2 ** 2 + 8 // 3 produce?

print(10 - 2 ** 2 + 8 // 3)   # 8
# Step 1: 2 ** 2 = 4    (exponent first)
# Step 2: 8 // 3 = 2    (floor division, same level as *, left-to-right)
# Step 3: 10 - 4 = 6    (subtraction)
# Step 4: 6 + 2 = 8     (addition)

The order is exponent → floor division → subtraction → addition. Same-level operators evaluate left-to-right; ** would be right-to-left if chained.

3. Division behaviors side by side.

print(10 / 2)        # 5.0  – float, not 5
print(7 / 2)         # 3.5
print(7 // 2)        # 3    – integer floor
print(-7 // 2)       # -4   – floor of -3.5 is -4, not -3
print(7 % 3)         # 1    – remainder of 7/3
print(-7 % 2)        # 1    – follows floor convention

The -7 // 2 = -4 trap is the most-tested division behavior on Python multiple-choice. Floor division rounds toward negative infinity, not toward zero.

4. Python and AP Pseudocode side by side.

# Python
remainder = 17 % 5     # 2
quotient  = 17 // 5    # 3
power     = 2 ** 10    # 1024
if (n % 2) == 0:
    print("even")

AP Pseudocode equivalent:

remainder <- 17 MOD 5
quotient  <- 17 / 5
power     <- 2 ^ 10
IF ((n MOD 2) = 0)
    DISPLAY("even")

Notice three differences: <- instead of = for assignment, ^ instead of ** for exponent, and the keyword MOD instead of %. Same math; different surface notation.

Common Pitfalls

  • Expecting an integer from /. 10 / 2 is 5.0, not 5. Use // for an integer quotient or wrap with int().
  • Forgetting that -7 // 2 is -4, not -3. Floor toward negative infinity. The Python multiple-choice trap.
  • Misreading ** chains. 2 ** 3 ** 2 is 512, not 64. Right-to-left associativity for exponentiation only.
  • Operator-precedence bugs in formulas. 100 - 10 / 100 * 200 is 80.0, not the intended 180.0. Add parentheses to express your intent: (100 - 10) / 100 * 200.
  • Mixing ** with *. 3 * 4 ** 2 is 48 (3 * 16), not (3*4)**2 = 144. Add parentheses if you mean the second.

Connect to the Exam

This section maps to Big Idea 3 (Algorithms and Programming), and its content surfaces on every exam form. Expect (a) precedence trace items where you must compute step-by-step what an expression evaluates to; (b) MOD items, especially for parity and cyclic indexing — n MOD 2 = 0 is the most-tested AP pattern; and (c) division-result items where you must distinguish / (float) from // (int floor). For the algorithm-tracing items in Unit 3, you will rely on the MOD and // behaviors established here. Memorize the even-check (n % 2 == 0) and the cyclic index pattern (index % length).

Self-Check

  • Without running Python, compute 3 + 4 * 2 - 1 and explain the order of operations step by step.
  • In AP Pseudocode, what does 13 MOD 4 evaluate to? Write the equivalent Python expression and the result.
  • Predict the output of -9 // 4 and explain in one sentence why it is not -2.
  • Compute 2 ** 3 ** 2 and explain why exponentiation associates right-to-left.
  • A student writes average = 80 + 90 + 70 / 3 expecting 80.0 but gets 193.33. Identify the bug and rewrite the line correctly.
