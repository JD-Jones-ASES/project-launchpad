AP CSP — Topic 4.1: Definite Loops

Core Question

How can a programmer instruct a computer to repeat the same block of code a fixed number of times without writing each repetition by hand? Definite loops solve this problem by separating the count of repetitions (specified once) from the work to be done. Both Python's for i in range(n) and AP Pseudocode's REPEAT n TIMES express this idea, and both ship with the same recurring trap: how the upper bound is interpreted. Mastering this section means controlling exactly how many times your loop runs and translating loop counts cleanly between Python and AP Pseudocode.

Key Concepts

  • Definite loop: Repeats a block of code a fixed, predetermined number of times. Terminates automatically once the count is reached.
  • Python for i in range(n): The standard definite loop. The loop variable i takes the values 0, 1, ..., n-1 — exactly n values, with n itself excluded.
  • range(a, b): Produces a, a+1, ..., b-1. Lower bound included; upper bound excluded.
  • range(a, b, step): Adds a custom increment. range(2, 11, 2) gives 2, 4, 6, 8, 10.
  • AP Pseudocode REPEAT n TIMES: Executes the body exactly n times. No loop variable is exposed; you must maintain a counter manually if the body needs the iteration index.
  • Off-by-one error: The single most common loop bug on the AP exam — caused by mistaking range(n) for range(n+1) or forgetting that range excludes the upper bound.
  • Translation rule for inclusion: To include a final value v in the iteration, the upper-bound argument to range must be v+1.
  • Nested for loops: A loop inside another loop. If the outer runs m times and the inner n times, the inner body runs m*n times.
  • Why nested loops are useful: They are the canonical pattern for processing 2D data — multiplication tables, image pixels, spreadsheet cells, 2D-list traversal in Unit 5.
  • Pioneer: Donald Knuth standardized loop-counting and asymptotic notation; his insight that a loop is a mathematical object you can analyze precisely underlies all algorithm-analysis questions on the exam.

Worked Examples / Code Walkthroughs

Example 1 — The off-by-one trap

# Want: print 1 through 10 inclusive
# Bug: stops at 9
for i in range(1, 10):
    print(i)

# Correct: include the value 10
for i in range(1, 11):
    print(i)

The fix: pass 11 as the upper bound. Since range excludes the top, you must write one more than your desired final value.

Example 2 — Python ↔ AP Pseudocode translation

for i in range(5):
    print(i * i)

REPEAT 5 TIMES
{
    DISPLAY(i * i)   <-- INVALID: i is not exposed by REPEAT
}

# Correct AP Pseudocode:
i <- 0
REPEAT 5 TIMES
{
    DISPLAY(i * i)
    i <- i + 1
}

The Pseudocode REPEAT counts silently, so any reference to the iteration index requires an explicit counter you maintain yourself.

Example 3 — Nested loop iteration count

for row in range(1, 4):       # 3 outer iterations: row = 1, 2, 3
    for col in range(1, 5):   # 4 inner iterations per outer: col = 1, 2, 3, 4
        print(row * col, end=' ')
    print()  # newline after each row

The inner print runs 3 * 4 = 12 times. Output:

1 2 3 4
2 4 6 8
3 6 9 12

This is the standard 2D-traversal pattern: outer loop over rows, inner loop over columns.

Example 4 — Stepping over even numbers

# Even numbers from 2 through 10 inclusive
for i in range(2, 11, 2):
    print(i)
# Output: 2, 4, 6, 8, 10

To include 10, the upper bound must be 11 — the same off-by-one rule applies even with a step argument.

Common Pitfalls

  • The off-by-one error (the headliner): range(1, N) produces 1 through N-1, missing N. To include N, write range(1, N+1). Always ask before writing range(a, b): "Does the problem want b included?"
  • Confusing range(n) with range(1, n): range(n) starts at 0; range(1, n) starts at 1. Both exclude n. Pick the start value the problem actually requires.
  • Trying to reference a Pseudocode loop variable: REPEAT n TIMES does not expose a counter. Inside the body, i is not defined. You must declare and increment your own counter.
  • Adding instead of multiplying nested counts: A 3x4 nested loop runs 12 times, not 7. The inner body runs once per (outer, inner) pair.
  • Forgetting the step argument in step-based iteration: range(2, 10, 2) stops at 8, not 10. Whenever you use a step, the upper bound rule still applies — push it one past the desired endpoint.

Connect to the Exam

Topic 4.1 is foundational for every algorithmic-analysis question on the AP exam. Translation between Python's for ... range(n) and AP Pseudocode's REPEAT n TIMES is tested directly, and indirectly through any question that traces a loop's output. Off-by-one errors are the single largest source of point loss on AP exam tracing questions — practice both directions of translation and always sanity-check by listing the first and last values the loop variable actually takes. Nested loops appear in 2D problems (Unit 5: lists) and in robot-grid problems (Topic 4.4), so your fluency with the iteration-count product (m*n) carries forward to more advanced sections.

Self-Check

  • Write a Python for loop that prints the multiples of 5 from 5 through 50 inclusive. Verify your upper bound by listing the first and last values the loop variable takes.
  • Translate for i in range(2, 6): print(i) into AP Pseudocode. Account for the missing loop variable by introducing your own counter.
  • A nested loop has outer range(1, 4) and inner range(0, 5). How many times does the inner body execute? Trace the (outer, inner) pairs to confirm.
  • A student writes range(0, N) when they wanted exactly N iterations starting at 1. Identify two distinct errors in this code and fix them.
  • Explain in your own words why range(n) and REPEAT n TIMES both run the body exactly n times even though their syntax differs. What design decision in Python explains the upper-bound exclusion?
