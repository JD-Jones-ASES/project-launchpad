AP CSP — Topic 7.2: Packet Switching and Routing

Core Question

How does the Internet move billions of pieces of data per second across the planet without melting under the load and without grinding to a halt when something breaks? The answer is packet switching: the sender's network stack chops every message into small, independently addressed packets, the network forwards them hop by hop, and the receiver reassembles them in order. This section unpacks the mechanism — what is in a packet, how routers decide where to send it, what happens when paths change or links congest — and shows why this design lets the Internet be simultaneously fast, scalable, and resilient.

Key Concepts

  • Packet: header (control info) + payload (data). Header carries source IP, destination IP, sequence number, checksum, and TTL.
  • Packet switching: each packet is routed independently; different packets in the same stream may take different physical routes; the receiver reassembles using sequence numbers.
  • Circuit switching: older model (still used by traditional phone network) where a dedicated circuit is reserved end-to-end for the duration of a session, even during silences.
  • Routing protocol: rules by which routers exchange topology information and update routing tables (OSPF inside one autonomous system, BGP between ISPs).
  • Routing table: a router's lookup map from destination IP ranges to the best outgoing link.
  • Hop: one segment of a packet's journey between two consecutive routers.
  • Bandwidth: max data rate, bits per second (Mbps, Gbps). Pipe-width metaphor.
  • Latency: time for a packet to traverse the network, usually round-trip time (RTT) in ms. Drive-time metaphor.
  • TTL (Time-to-Live): hop counter in the header; decremented at every router; packet is discarded when it hits zero. Prevents infinite loops.
  • Sequence number: header field marking each packet's position in the stream so the receiver can reassemble out-of-order packets.

Worked Examples / Mechanisms

  • The packet-path diagram. Two packets with the same source and destination may take entirely different routes. Packet 1 might go Source → R1 → R2 → R4 → Dst; packet 2 might go Source → R1 → R3 → R5 → Dst. Each router makes its forwarding decision independently based on current conditions; the receiver uses sequence numbers to reorder them.
  • Bandwidth conversion arithmetic. To transfer 800 MB over 100 Mbps at full utilization: convert MB to Mb (multiply by 8), then divide by the link rate. 800 MB × 8 = 6,400 Mb; 6,400 / 100 = 64 seconds. The trap is forgetting the byte-to-bit conversion and getting 8 seconds.
  • Bandwidth-vs-latency contrast. A satellite link rated 25 Mbps with 600 ms RTT can download files at acceptable throughput but is unusable for interactive gaming, because every player input must round-trip 600 ms before the game state updates. A local Ethernet LAN at 1 Gbps with 1 ms RTT serves both well. Bandwidth and latency are independent dimensions.
  • TTL loop prevention. A misconfigured routing loop sends a packet bouncing between three routers. With each hop, TTL decreases. Starting at 64 the packet survives 64 hops; on the 65th the TTL becomes 0 and the packet is discarded — preventing it from circulating forever.

Common Pitfalls

  • Confusing MB with Mb. 1 byte = 8 bits, so always multiply MB by 8 before dividing by Mbps. Forgetting this gives an answer 8x too small.
  • Treating bandwidth and latency as the same number. They are independent dimensions. A high-bandwidth link can have high latency (satellite); a low-bandwidth link can have low latency (a single fast Ethernet hop). Both matter, in different ways, for different workloads.
  • Assuming all packets in a stream follow the same route. Independent per-packet routing is the design. Out-of-order arrival is normal; sequence numbers handle reassembly.
  • Thinking circuit switching is "more reliable". Reserved circuits are simpler to reason about but use capacity inefficiently and crumble under heavy load. Packet switching is what enables billions of simultaneous conversations on shared capacity.
  • Believing TTL is a clock. TTL is a hop counter, not a wall-clock timer. It is decremented per router-hop, not per second.

Connect to the Exam

Section 7.2 supports both conceptual and quantitative AP-style questions. Expect items that ask you to (a) identify what makes packet switching different from circuit switching, (b) compute transfer time given bandwidth and file size (always check MB-to-Mb conversion), and (c) explain why dynamic routing and TTL combine to make the network resilient and loop-safe. Bandwidth-vs-latency is a perennial favorite for stimulus questions: "Given these two scenarios, which connection is appropriate for which task?" — the answer hinges on whether the workload is throughput-limited (bandwidth) or interactivity-limited (latency).

Self-Check

  • A video file is 800 MB. Over a 100 Mbps connection at full utilization, approximately how long (in seconds) will it take to transfer? Show the unit conversion.
  • Explain why packet switching allows the Internet to handle far more simultaneous users than circuit switching could. Use the words "shared capacity" in your answer.
  • A packet's TTL starts at 64 and is decremented at each router. What happens if a misconfiguration creates an infinite routing loop? Why is this mechanism important?
  • A satellite link offers 50 Mbps bandwidth and 700 ms round-trip latency. Identify one application for which this link is well-suited and one for which it is poorly suited. Justify each in one sentence.
  • Two consecutive packets from the same sender to the same receiver take different physical routes through the network. Explain how this is possible and how the receiver still delivers an ordered stream of bytes to the application.
