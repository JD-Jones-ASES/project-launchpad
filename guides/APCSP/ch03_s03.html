AP CSP — Topic 3.3: Conditional Execution

Core Question

How does a program choose among several alternatives — show one welcome message instead of another, classify a score as A or B or C, apply a child or adult or senior fare? The answer is the conditional statement: if, elif, and else. This section walks through Python's syntax (colons, indentation, the elif keyword) and contrasts it carefully with AP Pseudocode's brace-delimited blocks. It also explores why ORDER MATTERS in an if/elif chain and what distinguishes a syntax error (Python crashes) from a logic error (Python runs and gives wrong answers).

Key Concepts

  • Conditional statement: A construct that evaluates a Boolean expression and runs one of several code blocks. Exactly one branch executes per run.
  • if (Python): Tests a condition; runs an indented body when the condition is True. The colon : after the condition is mandatory.
  • else: Optional fallback; runs when the if condition is False. Exactly one of if/else bodies executes.
  • elif: Short for "else if." Inserts intermediate branches. Python tests them top-to-bottom and STOPS at the first True condition.
  • AP Pseudocode equivalents: IF, ELSE IF, ELSE (uppercase, two-word ELSE IF). Blocks are delimited by { }. No colon after the condition.
  • Indentation in Python: Defines block membership. Inconsistent indentation -> IndentationError; misleading-but-valid indentation -> silent logic error.
  • Order matters: Because Python stops at the first True condition, reversing branch order can change which branch fires (and produce a different classification).
  • Syntax error vs logic error: A syntax error (missing colon, bad indentation) prevents the program from running. A logic error allows the program to run but produces wrong output. Logic errors are harder to find.
  • Boundary care: >= 12 includes 12; > 12 excludes it. Off-by-one mistakes at boundaries are a frequent bug.
  • One branch only: Once a branch fires, the rest of the chain is skipped — even branches that would also have been True.

Worked Examples / Code Walkthroughs

Example 1 — Letter grade classifier (Python)

score = 83

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
elif score >= 60:
    grade = "D"
else:
    grade = "F"

print(f"Score: {score}, Grade: {grade}")  # Score: 83, Grade: B

Python evaluates from top to bottom. 83 >= 90 is False (skip). 83 >= 80 is True — grade is assigned "B" and the rest of the chain is skipped, even though 83 >= 70 would also be True.

Example 2 — Same logic in AP Pseudocode

IF (score >= 90)
{
    grade <- "A"
}
ELSE IF (score >= 80)
{
    grade <- "B"
}
ELSE IF (score >= 70)
{
    grade <- "C"
}
ELSE
{
    grade <- "F"
}

Note the differences from Python: uppercase keywords, ELSE IF as two words, curly braces around blocks, no colon, arrow <- for assignment.

Example 3 — The order trap

If we reverse the chain to ascending order:

if score >= 60:        # First check
    grade = "D"
elif score >= 70:
    grade = "C"
# ...

Now a score of 83 would match the very first condition (83 >= 60 is True), get "D", and the remaining elifs are never tested. Order is semantically significant in an if/elif chain.

Example 4 — Indentation-driven logic error

temperature = 35

# CORRECT: both prints are inside the if-block:
if temperature > 30:
    print("Hot day!")
    print("Wear sunscreen.")  # also inside if-block

# MISLEADING: second print is OUTSIDE the if:
if temperature > 30:
    print("Hot day!")
print("Always wear sunscreen.")  # runs every time, regardless of temperature!

Both versions are syntactically valid — both run without raising an error. The "misleading" version produces wrong output (sunscreen advice on a 5-degree day). This is the classic logic error: no crash, no warning, just incorrect behavior.

Common Pitfalls

  • Forgetting the colon: if x > 0 without a colon raises SyntaxError. Every Python if, elif, and else line must end with :.
  • Inconsistent indentation: Mixing tabs and spaces, or indenting two statements differently, raises IndentationError.
  • Using else if (two words) in Python: Python's keyword is elif (one word). Two words is a syntax error.
  • Putting branches in wrong order: Reversing thresholds in a tiered classifier silently changes results. Always test boundary values.
  • Confusing = with ==: if x = 5: raises SyntaxError; equality comparison is ==.
  • Off-by-one at boundaries: age > 12 and age < 18 excludes ages 12 and 18; age >= 12 and age <= 17 includes both. Choose carefully.

Connect to the Exam

Topic 3.3 is foundational under Big Idea 3 (Algorithms and Programming). AP exam questions frequently include a 5-10 line if/elif/else snippet and ask "what does this print when score = 75?" or "what happens if you reverse the order of two elif branches?" The official Pseudocode reference uses uppercase IF/ELSE/ELSE IF with curly braces — be ready to translate between Python and Pseudocode in either direction. Watch for boundary conditions: >= versus > is a near-universal source of distractor answers in tiered-classifier questions.

Self-Check

  • Trace the grade classifier with score = 70. Which branch fires, and what is grade?
  • Translate this Python snippet into AP Pseudocode: if x > 0: print("positive")\nelse: print("non-positive").
  • A programmer writes if age > 12 and age < 18: for the student tier. For which age value does this differ from age >= 12 and age <= 17, and what wrong outcome results?
  • Why does reversing the if/elif branches in a tiered classifier (lowest threshold first) usually break the logic?
  • What is the difference between an IndentationError and a logic error? Give a concrete example of each from the section.
