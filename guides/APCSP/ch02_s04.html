AP CSP — Topic 2.4: Strings and System I/O

Core Question

How does a Python program communicate with a human user, and what catches every beginner off guard the first time? print() sends text to the screen, input() reads text typed at the keyboard. Together they are the input/output (I/O) interface every interactive program needs. The trap that every Unit 2 student hits — and that the AP exam tests — is that input() always returns a string, no matter what the user types. To do arithmetic on a numeric input, you must convert with int() or float() explicitly. This section also introduces string concatenation, repetition, length, and the modern f-string syntax for embedding values into messages.

Key Concepts

  • print(): Sends arguments to standard output, separated by spaces and followed by a newline. Optional sep keyword overrides the separator; optional end keyword overrides the trailing newline. print() with no arguments prints a blank line.
  • input(): Displays an optional prompt and reads one line of text from the user. Always returns a str, even when the user types digits. To use the value in arithmetic, wrap with int() (whole numbers) or float() (decimals).
  • int(x), float(x), str(x): Type-conversion functions. int("42") is 42; int(3.9) is 3 (truncation, not rounding); int("hello") raises ValueError. float("3.14") is 3.14. str(42) is "42" — needed when concatenating a number into a sentence.
  • Concatenation (+): Joins two strings end-to-end. Both operands must be strings; mixing string and int with + raises TypeError.
  • Repetition (*): Repeats a string a given number of times: "Ha" * 3 is "HaHaHa". One operand string, one integer. Useful for printing dividers ("=" * 20).
  • len(): Returns the number of characters in a string, including spaces and punctuation. len("Hello, World!") is 13.
  • f-strings: Preferred modern syntax. Prefix the string with f and embed expressions in {}. Python converts embedded values automatically — no str() needed inside.
  • AP Pseudocode I/O: name <- INPUT() reads from user; DISPLAY("Hello, " + name) prints. Note INPUT() takes no prompt argument; the exam describes the typed input in surrounding narrative.

Worked Examples / Code Walkthroughs

1. The input() type trap — buggy and fixed.

# BUGGY: input() returns a string
count = input("How many apples? ")   # user types: 5
total = count + 3                    # TypeError!

# CORRECT: convert before arithmetic
count = int(input("How many apples? "))   # int("5") -> 5
total = count + 3                          # 5 + 3 = 8
print("Total:", total)                     # Total: 8

The error message — TypeError: can only concatenate str (not "int") to str — is what Python raises when + finds incompatible operand types. The fix is to wrap input() in int() (or float() for decimals).

2. Building a sentence two ways.

score = 95

# Old style: explicit str() conversion with concatenation
message = "Score: " + str(score) + " out of 100."
print(message)        # Score: 95 out of 100.

# Modern style: f-string
message = f"Score: {score} out of 100."
print(message)        # Score: 95 out of 100.

f-strings are shorter, clearer, and perform automatic conversion — preferred in modern Python.

3. The Greeter program lab walk-through. Fill in the missing lines:

name = input("What is your name? ")
favorite = int(input("What is your favorite number? "))   # convert to int

print("Hello,", name + "!")
print(f"Your favorite number is {favorite}.")
print(f"Twice your favorite number is {favorite * 2}.")

If you forgot the int() wrap, the line favorite * 2 on a string would print the string repeated (e.g., "77" for input 7) instead of 14.

4. Python and AP Pseudocode side by side.

# Python
name = input("Enter name: ")
age  = int(input("Enter age: "))
print("Hello,", name)
print("Age:", age)

AP Pseudocode equivalent:

name <- INPUT()
age  <- INPUT()
DISPLAY("Hello, " + name)
DISPLAY("Age: " + age)

INPUT() takes no prompt; type conversion is not shown in pseudocode — the exam assumes consistent typing in surrounding narrative.

Common Pitfalls

  • Forgetting the int() wrap on input(). count + 3 on a string input crashes with TypeError. Always wrap when arithmetic follows.
  • Trying to concatenate a number into a string with +. "Score: " + 95 raises TypeError. Use str(95) or an f-string.
  • Confusing string repetition with arithmetic. "3" * 5 is "33333" (repetition); 3 * 5 is 15 (arithmetic). The operands' types decide the meaning.
  • Calling int("3.5"). This raises ValueError because the string contains a decimal point. Use float("3.5") or int(float("3.5")).
  • Inconsistent quotes inside f-strings. Don't reuse the outer quote character inside {}. If the f-string is double-quoted, use single quotes for any string literal inside the braces.

Connect to the Exam

This section maps to Big Idea 1 (Creative Development) for the I/O patterns and Big Idea 3 (Algorithms) for the algorithmic uses of conversion. Expect (a) AP Pseudocode items asking you to translate INPUT() / DISPLAY() to and from Python, (b) trace items where you predict the output of an input() followed by a string operation, (c) bug-finding items where the omitted int() wrap is the cause, and (d) f-string vs concatenation items asking which produces the same output. The string-vs-integer trap from Section 2.2 is reinforced here every time a user types a number — internalize the rule that input() returns a string always.

Self-Check

  • Write one line of Python that reads the user's age as an integer stored in a variable called age.
  • Predict the output of print(x + x) after x = input("Enter a number: ") if the user types 3. Explain in one sentence why the answer is not 6.
  • Translate print("Hello,", name) into AP Pseudocode using DISPLAY().
  • Rewrite this concatenation as an f-string: "You are " + str(age) + " years old".
  • A student writes int(input("Enter price: ")) but the user types 4.99. What error category is raised, and what is the fix?
