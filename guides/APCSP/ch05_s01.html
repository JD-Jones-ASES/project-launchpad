AP CSP — Topic 5.1: Introduction to Lists

Core Question

How does a single named list let a program manage data that would otherwise require a separate variable for every value? Lists are the foundation of every aggregate computation in Unit 5 and beyond, and the recurring Study Streak Tracker — built incrementally across Units 5 and 6 and submitted as a Create PT artifact in Unit 9 — uses lists to scale from three sessions to a full semester without rewriting any code.

Key Concepts

  • List: An ordered, mutable sequence of values in square brackets, capable of holding any type and any number of elements. Length changes at runtime through append, remove, or reassignment.
  • Empty list: Created with study_records = []. Has zero elements but is ready to receive them via .append().
  • append: list.append(item) adds item to the end, increasing length by one. Modifies the list in place and returns None. Successive appends fill index 0, then 1, then 2.
  • Index: A numeric position. Python uses zero-based indexing (first element at index 0, last at len(lst) - 1). AP Pseudocode uses one-based indexing (first element at index 1, last at LENGTH(lst)).
  • Negative indexing (Python only): lst[-1] is the last element, lst[-2] the second-to-last. AP Pseudocode has no negative indexing; translate lst[-1] to lst[LENGTH(lst)].
  • for loop: for record in study_records: visits every element in order. The loop variable receives the element's value directly.
  • Tuple: An ordered, immutable grouping of related fields, written with parentheses. (date, subject, minutes) groups three values into one record.
  • List vs. tuple: Lists are mutable; tuples are immutable. A list of tuples is a common design: the outer list grows with new records, while each inner tuple is a fixed snapshot of one record.
  • Lists store references: A Python list is an array of pointers to objects, which is why lists can hold mixed types and why lst[i] is always constant time.
  • One name for many items: Lists let one variable name represent any number of related values. The for-loop body is the same whether the list has 3 elements or 300.

Worked Examples / Code Walkthroughs

Building the Study Streak Tracker incrementally

study_records = []                                       # empty list, ready to grow
study_records.append(("2025-09-01", "math", 45))         # index 0
study_records.append(("2025-09-01", "history", 30))      # index 1
study_records.append(("2025-09-02", "math", 60))         # index 2
print(len(study_records))   # 3
print(study_records[1])     # ("2025-09-01", "history", 30)

Each .append() adds exactly one tuple to the end. Index 1 is the second element because Python is zero-indexed.

Iterating with tuple unpacking

for record in study_records:
    date, subject, minutes = record
    print(date, subject, minutes)

The loop visits every record in insertion order. Unpacking binds three names to the tuple's three fields so the body reads naturally.

Tuple immutability vs. list mutability

session = ("2025-09-01", "math", 45)
# session[2] = 50          # TypeError: 'tuple' object does not support item assignment

subjects = ["math", "history"]
subjects[0] = "physics"     # OK
print(subjects)             # ['physics', 'history']

Tuples freeze a record's fields; lists let you swap elements freely. The Study Streak Tracker uses tuples inside a list precisely so individual sessions cannot be silently rewritten while the outer log can still grow.

Indexing both ends

subjects = ["math", "history", "science", "art"]
print(subjects[0])    # math   (first)
print(subjects[3])    # art    (last in zero-indexed terms)
print(subjects[-1])   # art    (last via negative index)
print(len(subjects))  # 4

In AP Pseudocode, the last element would be subjects[LENGTH(subjects)] — there is no -1 shortcut.

Common Pitfalls

  • Off-by-one when indexing: Python's first element is lst[0], not lst[1]. AP Pseudocode reverses this. The error is so common that the entire next section (5.2) is devoted to it.
  • Treating .append() as a return value: x = study_records.append(record) sets x to None. Append modifies in place; do not assign its result.
  • Trying to mutate a tuple: session[2] = 50 raises TypeError. If a value needs to change, the record must be replaced as a whole, or the design must use a list instead of a tuple.
  • Using negative indexing in Pseudocode: lst[-1] is Python only. On the AP exam, the equivalent Pseudocode is lst[LENGTH(lst)].
  • Confusing len() with the largest valid index: For a list of length n, valid Python indices are 0 to n - 1. Writing lst[len(lst)] raises IndexError.

Connect to the Exam

AP CSP exam questions on lists almost always test indexing, traversal, and the cost of growing a list. Multiple-choice items ask you to predict the value of lst[i] after a sequence of appends, or to read a short for loop and identify which records the body processes. The Create Performance Task awards a Data Abstraction rubric point that requires you to identify a list in your program and explain what each element represents — the Study Streak Tracker introduced in this section is the running example used to set up that rubric response. Whenever you see an AP item that mixes Python and Pseudocode lists, annotate the index convention before tracing.

Self-Check

  1. Write two Python statements: the first creates an empty list called temps; the second appends the integer 72. What is the value of temps[0]?
  2. If colors = ["red", "green", "blue"], what does colors[-1] evaluate to in Python? What is the equivalent AP Pseudocode expression?
  3. Explain in one sentence why using a list is better than using three separate variables record1, record2, record3 to store three study sessions.
  4. Trace the value of study_records[2] after three append calls of (date, subject, minutes) tuples in the order shown in the Study Streak Tracker example.
  5. Why does the Study Streak Tracker use a tuple for each individual session rather than another list?
