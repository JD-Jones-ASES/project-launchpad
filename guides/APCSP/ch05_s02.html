AP CSP — Topic 5.2: The Indexing Dilemma

Core Question

Why does the same list have two completely different sets of valid indices depending on the programming language, and how do you keep the two systems straight when tracing algorithms by hand? Python uses zero-based indexing while AP Pseudocode uses one-based indexing. Chief Reader reports identify this single discrepancy as the largest source of point loss on AP CSP algorithm-tracing questions, year after year. The defensive technique you build in this section saves those points on exam day.

Key Concepts

  • Zero-based indexing (Python): The first element is at index 0, the second at 1, the last at len(lst) - 1. lst[len(lst)] raises IndexError. Used by Python, C, C++, Java, JavaScript, C#.
  • One-based indexing (AP Pseudocode): The first element is at index 1, the second at 2, the last at LENGTH(lst). lst[0] is invalid Pseudocode. Used by AP Pseudocode, MATLAB, Lua, R.
  • The off-by-one rule: Every index shifts by one when translating. Python lst[i] is Pseudocode lst[i + 1], and Pseudocode lst[i] is Python lst[i - 1].
  • Loop bound — Python: range(len(lst)) generates 0 to len(lst) - 1 inclusive. The condition while i < n stops BEFORE i reaches n.
  • Loop bound — Pseudocode: FOR EACH i IN [1, LENGTH(lst)] generates 1 to LENGTH(lst) inclusive. The condition REPEAT UNTIL (i > n) stops AFTER processing i = n.
  • Inverted condition: while X (continue while true) and REPEAT UNTIL X (stop when true) negate each other. Translating a loop bound also requires flipping the Boolean.
  • Negative indexing (Python only): lst[-1] is the last, lst[-2] is the second-to-last. Pseudocode has no negative indices; replace lst[-1] with lst[LENGTH(lst)].
  • Why two systems?: Zero-based indexing matches address arithmetic in C-family languages. One-based indexing matches how humans describe sequences ('first item, second item') and reduces cognitive load for novices — the College Board's choice for Pseudocode.
  • Annotation discipline: Always write the language at the top of your trace and label every index access. Never assume the convention — state it.
  • Same data, different labels: A list of three strings has the same VALUES whether indexed Python-style or Pseudocode-style; only the index NUMBERS change.

Worked Examples / Code Walkthroughs

Side-by-side: same list, two index conventions

# Python - 0-based indexing
subjects = ["math", "history", "science", "art"]
# Index:    0       1          2          3
print(subjects[0])    # math   (FIRST)
print(subjects[3])    # art    (LAST, len-1)
print(subjects[-1])   # art    (LAST, negative)

// AP Pseudocode - 1-based indexing
subjects <- ["math", "history", "science", "art"]
// Index:    1       2          3          4
DISPLAY(subjects[1])              // math (FIRST)
DISPLAY(subjects[LENGTH(subjects)]) // art (LAST)
// subjects[-1] is INVALID in Pseudocode

The list has the same four strings. Only the index numbers shift by one.

Annotated trace: summing four numbers

# Python - 0-based indexing
scores = [85, 92, 78, 95]
# Index:   0   1   2   3        (Python indices, 0 to len-1)

total = 0
for i in range(len(scores)):     # i takes values 0, 1, 2, 3
    total = total + scores[i]    # scores[0]=85, scores[1]=92, ...
print(total)                     # Output: 350

// AP Pseudocode - 1-based indexing
scores <- [85, 92, 78, 95]
// Index:   1   2   3   4        (Pseudo indices, 1 to LENGTH)

total <- 0
FOR EACH i IN [1, LENGTH(scores)]   // i takes values 1, 2, 3, 4
    total <- total + scores[i]      // scores[1]=85, scores[2]=92, ...
DISPLAY(total)                       // Output: 350

Same algorithm, same result (350), completely different index labels. The annotation # Index: 0 1 2 3 vs // Index: 1 2 3 4 is the discipline that prevents mid-trace switching.

Translating a while loop

# Python
lst = [10, 20, 30, 40]
n = len(lst)
i = 0
while i < n:        # continue WHILE true
    print(lst[i])
    i = i + 1
# loop ends when i = 4 (4 < 4 is False)

// AP Pseudocode
lst <- [10, 20, 30, 40]
n <- LENGTH(lst)
i <- 1
REPEAT UNTIL (i > n)   // stop WHEN this becomes true
    DISPLAY(lst[i])
    i <- i + 1
// loop ends when i = 5 (5 > 4 is True)

Two independent shifts: index start (0 to 1), and Boolean polarity (i < n becomes i > n).

Negative-index translation

# Python: print the last element
print(lst[-1])

// AP Pseudocode: print the last element
DISPLAY(lst[LENGTH(lst)])

Writing lst[-1] in Pseudocode is invalid and earns zero credit on the AP exam.

Common Pitfalls

  • Mentally substituting range(len(L)) for [1, LENGTH(L)] without shifting: Forces a one-step skip backward, producing a shifted (wrong) answer. The annotation habit prevents this.
  • Writing lst[0] in Pseudocode: Index 0 does not exist in Pseudocode. The first element is lst[1].
  • Writing lst[LENGTH(lst)] in Python: That index is one past the end and raises IndexError. In Python the last index is len(lst) - 1.
  • Translating lst[-1] verbatim into Pseudocode: Negative indexing is Python-only. The Pseudocode equivalent is lst[LENGTH(lst)].
  • Forgetting to flip while-condition polarity: Python's while i < n inverts to Pseudocode's REPEAT UNTIL (i > n), not the same condition with > swapped for <.

Connect to the Exam

Algorithm-tracing items on the AP CSP exam routinely mix Python and Pseudocode list operations in the same problem. The Chief Reader specifically identifies the 0/1 indexing translation as the largest single source of lost points across years of administrations. Whenever a stimulus uses Pseudocode, write '1-based' at the top of your work; whenever it uses Python, write '0-based'. Label every lst[i] you reference. The minute you save by skipping annotation is the minute that costs the point.

Self-Check

  1. A Pseudocode algorithm begins lst <- [5, 10, 15, 20]. What is lst[1]? What is lst[LENGTH(lst)]?
  2. Translate the Python loop for i in range(len(scores)): total = total + scores[i] to AP Pseudocode. Annotate both versions with the index convention used.
  3. A student translates lst[-1] to Pseudocode as lst[-1]. Explain the error and write the correct Pseudocode expression.
  4. Identify the FIRST and LAST valid index of a Python list of length 6, and the same for an AP Pseudocode list of length 6.
  5. Why is REPEAT UNTIL (i > n) the correct Pseudocode equivalent of Python's while i < n? Explain the two independent shifts (index start and Boolean polarity) involved.
