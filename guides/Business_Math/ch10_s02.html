Business Applications: Optimization in Business

Overview

This section introduces linear programming as a powerful tool for businesses to make optimal decisions when faced with limited resources. It guides you through the process of setting up and solving real-world optimization problems, demonstrating how to maximize profit or revenue and minimize costs across various business scenarios like manufacturing, food service, and logistics.

Key Concepts

  • Optimization in Business: The process of finding the best possible outcome (e.g., maximum profit, minimum cost) given a set of limited resources and specific constraints.
  • Linear Programming (LP): A mathematical method used to determine the optimal solution for problems that can be expressed as a linear objective function subject to a system of linear inequalities (constraints).
  • Objective Function: A linear equation that represents the quantity to be maximized (e.g., profit, revenue) or minimized (e.g., cost). It is typically denoted by Z.
  • Constraints: Linear inequalities that represent the limitations or restrictions on the resources available or requirements that must be met (e.g., labor hours, raw materials, production capacity, nutrient requirements).
  • Non-negativity Constraints: A common type of constraint ($x \ge 0, y \ge 0$) ensuring that the quantity of items produced or used cannot be negative.
  • Feasible Region: The graphical area defined by the intersection of all constraints. Any point within this region represents a possible (feasible) solution that satisfies all limitations.
  • Corner Point Theorem: States that the optimal solution (maximum or minimum value of the objective function) for a linear programming problem, if it exists, will always occur at one of the corner points (vertices) of the feasible region.
  • Unbounded Feasible Region: A feasible region that extends indefinitely in one or more directions. For minimization problems, an optimal solution might still exist at a corner point, but for maximization problems, an unbounded region may imply no maximum value.
  • Multiple Optimal Solutions: Occurs when the objective function's optimal value is achieved at two or more adjacent corner points. In such cases, every point along the line segment connecting these corner points also yields the same optimal value.

Important Terms

  • Linear Programming: A mathematical technique for solving optimization problems where the objective function and constraints are all linear.
  • Optimization: The process of finding the most effective or favorable solution to a problem.
  • Objective Function: The mathematical expression (a linear equation) that describes the quantity to be maximized or minimized.
  • Constraints: The limiting conditions or restrictions expressed as linear inequalities that must be satisfied by the variables in a linear programming problem.
  • Feasible Region: The set of all points that satisfy all the constraints of a linear programming problem.
  • Corner Points (Vertices): The points where the boundary lines of the feasible region intersect. These are crucial because the optimal solution always lies at one of these points.
  • Resource Allocation: The process of assigning available resources to various uses to achieve specific objectives.

Detailed Notes

The Seven-Step Process for Solving Linear Programming Problems

All linear programming problems in business applications follow a systematic approach:

  1. Define the variables: Clearly identify what the unknown quantities are and assign variables (e.g., $x$, $y$) to them. These usually represent the number of units of each product to produce, or the amount of each ingredient to use.
  2. Write the objective function: Formulate a linear equation (Z) that represents the quantity you want to optimize (maximize profit/revenue or minimize cost) in terms of your defined variables.
  3. Write the constraints: Translate all resource limitations, requirements, and conditions into a system of linear inequalities. Always include non-negativity constraints ($x \ge 0, y \ge 0$) as quantities cannot be negative. Simplify constraints if possible.
  4. Graph the feasible region: Plot the boundary lines for each constraint. Determine the region that satisfies all inequalities simultaneously. This region represents all possible solutions.
    • For "less than or equal to" ($\le$) constraints, shade below the line.
    • For "greater than or equal to" ($\ge$) constraints, shade above the line.
    • Non-negativity constraints ($x \ge 0, y \ge 0$) restrict the feasible region to the first quadrant.
  5. Find the corner points: Identify the coordinates of all vertices (corner points) of the feasible region. These points are typically intersections of the boundary lines.
  6. Evaluate the objective function: Substitute the coordinates of each corner point into the objective function (Z).
  7. State the solution: Based on the evaluation, determine which corner point yields the optimal value (the maximum for maximization problems, or the minimum for minimization problems). Clearly state the optimal values for the variables and the resulting maximum/minimum value of the objective function.

Example 1: Maximize Profit — Woodcraft Custom Furniture

  • Goal: Maximize profit for tables ($x$) and chairs ($y$).
  • Objective Function: $Z = 80x + 50y$
  • Constraints:
    • Labor: $6x + 3y \le 120 \Rightarrow 2x + y \le 40$
    • Lumber: $30x + 10y \le 450 \Rightarrow 3x + y \le 45$
    • Non-negativity: $x \ge 0, y \ge 0$
  • Feasible Region: Bounded region in the first quadrant, below both simplified constraint lines.
  • Corner Points & Evaluation:
    • $(0, 0) \Rightarrow Z = 80(0) + 50(0) = 0$
    • $(15, 0) \Rightarrow Z = 80(15) + 50(0) = 1200$
    • $(5, 30)$ (intersection of $2x + y = 40$ and $3x + y = 45$) $\Rightarrow Z = 80(5) + 50(30) = 400 + 1500 = 1900$
    • $(0, 40) \Rightarrow Z = 80(0) + 50(40) = 2000$ (Maximum)
  • Solution: Produce 0 tables and 40 chairs for a maximum profit of $2,000.00 per week. This highlights that intuition isn't always best; linear programming shows that focusing solely on chairs (lower per-unit profit but much lower resource consumption) is more profitable here.

Example 2: Minimize Cost — GreenGrow Fertilizer

  • Goal: Minimize cost for Mix A ($x$) and Mix B ($y$).
  • Objective Function: $Z = 3x + 4y$
  • Constraints:
    • Nitrogen: $4x + 2y \ge 12 \Rightarrow 2x + y \ge 6$
    • Phosphorus: $x + 2y \ge 8$
    • Non-negativity: $x \ge 0, y \ge 0$
  • Feasible Region: Unbounded region in the first quadrant, above both simplified constraint lines. When the feasible region is unbounded, a minimum (but not necessarily a maximum) can still exist at a corner point.
  • Corner Points & Evaluation:
    • $(8, 0)$ (intersection of $x + 2y = 8$ and $y=0$) $\Rightarrow Z = 3(8) + 4(0) = 24$
    • $(4/3, 10/3)$ (intersection of $2x + y = 6$ and $x + 2y = 8$) $\Rightarrow Z = 3(4/3) + 4(10/3) = 4 + 40/3 = 12/3 + 40/3 = 52/3 \approx 17.33$ (Minimum)
    • $(0, 6)$ (intersection of $2x + y = 6$ and $x=0$) $\Rightarrow Z = 3(0) + 4(6) = 24$
  • Solution: Use approximately $1.33$ pounds of Mix A and $3.33$ pounds of Mix B (exactly $4/3$ lb A and $10/3$ lb B) to achieve a minimum cost of $17.33 per bag.
  • Quick Practice Verification: Optimal blend provides $4(4/3) + 2(10/3) = 16/3 + 20/3 = 36/3 = 12$ units of nitrogen and $4/3 + 2(10/3) = 4/3 + 20/3 = 24/3 = 8$ units of phosphorus, exactly meeting the requirements.

Example 3: Maximize Revenue — Sweet Sunrise Bakery

  • Goal: Maximize revenue for boxes of cookies ($x$) and cakes ($y$).
  • Objective Function: $Z = 12x + 18y$
  • Constraints:
    • Flour: $2x + 3y \le 180$
    • Sugar: $x + 2y \le 100$
    • Oven Capacity: $y \le 40$
    • Non-negativity: $x \ge 0, y \ge 0$
  • Feasible Region: Bounded by five lines (including axes), forming a polygon in the first quadrant.
  • Corner Points & Evaluation:
    • $(0, 0) \Rightarrow Z = 0$
    • $(90, 0)$ (intersection of $2x + 3y = 180$ and $y = 0$) $\Rightarrow Z = 12(90) + 18(0) = 1080$
    • $(60, 20)$ (intersection of $2x + 3y = 180$ and $x + 2y = 100$) $\Rightarrow Z = 12(60) + 18(20) = 720 + 360 = 1080$
    • $(20, 40)$ (intersection of $x + 2y = 100$ and $y = 40$) $\Rightarrow Z = 12(20) + 18(40) = 240 + 720 = 960$
    • $(0, 40)$ (intersection of $x = 0$ and $y = 40$) $\Rightarrow Z = 12(0) + 18(40) = 720$
  • Solution: The maximum revenue is $1,080.00 per day. This maximum occurs at two distinct corner points: $(90, 0)$ and $(60, 20)$. This indicates multiple optimal solutions, meaning any combination of cookies and cakes along the line segment connecting $(90, 0)$ and $(60, 20)$ will also yield the same maximum revenue. Priya can choose based on other factors like product variety.

Example 4: Minimize Cost — QuickShip Logistics

  • Goal: Minimize daily operating cost for small vans ($x$) and large trucks ($y$).
  • Objective Function: $Z = 200x + 450y$
  • Constraints:
    • Package Capacity: $50x + 150y \ge 600 \Rightarrow x + 3y \ge 12$
    • Drivers: $x + y \le 10$
    • Large Trucks Available: $y \le 4$
    • Non-negativity: $x \ge 0, y \ge 0$
  • Feasible Region: Bounded region defined by the intersection of four lines in the first quadrant.
  • Corner Points & Evaluation:
    • $(9, 1)$ (intersection of $x + 3y = 12$ and $x + y = 10$) $\Rightarrow Z = 200(9) + 450(1) = 1800 + 450 = 2250$
    • $(0, 4)$ (intersection of $x + 3y = 12$ and $y = 4$) $\Rightarrow Z = 200(0) + 450(4) = 0 + 1800 = 1800$ (Minimum)
    • $(6, 4)$ (intersection of $x + y = 10$ and $y = 4$) $\Rightarrow Z = 200(6) + 450(4) = 1200 + 1800 = 3000$
  • Solution: The minimum daily operating cost is $1,800.00, achieved by using 0 small vans and 4 large trucks.
  • Quick Practice Verification: $50(0) + 150(4) = 600$ packages, exactly meeting the minimum requirement.

Historical Context

Linear programming was developed during World War II to efficiently manage military logistics. Today, it's widely used across industries, from small businesses to large corporations, for optimizing production, scheduling, and resource allocation.
