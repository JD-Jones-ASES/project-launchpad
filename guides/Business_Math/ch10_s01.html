Study Guide: Graphing Inequalities and Finding Feasible Regions

Overview

This section introduces the fundamental mathematical tools for linear programming: working with linear inequalities in two variables. You will learn how to graph these inequalities on a coordinate plane, identify regions that satisfy multiple constraints (called feasible regions), and find the crucial "corner points" of these regions. This understanding is essential for setting up and solving real-world business optimization problems, where you aim to maximize profit or minimize cost within defined limits.

Key Concepts

  • Linear Inequalities in Two Variables: Unlike equations, inequalities represent a range of solutions, graphically shown as a shaded region on a coordinate plane.
  • Graphing a Single Linear Inequality: Involves graphing a boundary line (solid for $\le, \ge$; dashed for $<, >$) and using a test point (like the origin) to determine which side of the line to shade.
  • Systems of Linear Inequalities: Multiple inequalities considered together, forming a set of constraints.
  • Feasible Region: The area on the graph where all inequalities in a system are simultaneously satisfied. This is the overlapping shaded region.
  • Non-Negativity Constraints: In business applications, variables like production quantities or hours cannot be negative, so $x \ge 0$ and $y \ge 0$ are almost always included, restricting the feasible region to the first quadrant.
  • Corner Point (Vertex): A point where two or more boundary lines of the feasible region intersect. These points are critical for optimization.
  • Objective Function: A linear expression ($Z = ax + by$) that represents the quantity you want to maximize (e.g., profit) or minimize (e.g., cost).
  • Linear Programming: A mathematical method used to find the optimal (maximum or minimum) value of an objective function subject to a set of linear constraints (inequalities).
  • Corner Point Theorem: States that if an optimal solution exists for a linear programming problem, it will always occur at one of the corner points of the feasible region. This simplifies the search for the optimal solution dramatically.

Important Terms

  • Linear Inequality in Two Variables: A mathematical statement in the form $ax + by < c$, $ax + by \le c$, $ax + by > c$, or $ax + by \ge c$, where $a, b, c$ are real numbers and $x, y$ are variables.
  • Solution (of an Inequality): Any ordered pair $(x, y)$ that makes the inequality true.
  • System of Linear Inequalities: Two or more linear inequalities considered together.
  • Feasible Region: The set of all points that satisfy every inequality in a system simultaneously; it is the solution set for the system.
  • Corner Point (Vertex): A point on the boundary of the feasible region where two or more boundary lines intersect.
  • Objective Function: A linear expression, typically $Z = ax + by$, that a linear programming problem seeks to maximize or minimize.
  • Linear Programming: A mathematical technique used to determine the optimal allocation of resources by maximizing or minimizing a linear objective function subject to linear constraints.
  • Non-negativity Constraints: The inequalities $x \ge 0$ and $y \ge 0$, which ensure that decision variables represent non-negative quantities, commonly used in real-world applications.

Detailed Notes

Introduction to Inequalities

Unlike equations, which have one or a finite number of solutions, linear inequalities in two variables describe a whole region of solutions. These regions are crucial for business applications where resources are limited and decisions need to optimize outcomes (e.g., maximize profit, minimize cost).

Graphing a Single Linear Inequality

To visualize the infinite solutions of an inequality like $ax + by \le c$, we graph it on a coordinate plane.

  1. Graph the Boundary Line:

    • Temporarily replace the inequality symbol with an equals sign ($=$) to get the equation of the boundary line.
    • Solid Line: Use for $\le$ or $\ge$. This indicates that points on the line are part of the solution set.
    • Dashed Line: Use for $<$ or $>$. This indicates that points on the line are not part of the solution set.
    • Tip: Find the x- and y-intercepts to easily plot the line.

  2. Choose a Test Point:

    • Pick any point not on the boundary line. The origin $(0, 0)$ is usually the easiest choice, unless the line passes through it. If it does, choose another simple point like $(1, 0)$ or $(0, 1)$.

  3. Substitute and Shade:

    • Substitute the coordinates of the test point into the original inequality.
    • If True: Shade the side of the boundary line that contains the test point.
    • If False: Shade the side of the boundary line that does not contain the test point.

Example: Graph $2x + y \le 6$

  1. Boundary Line: $2x + y = 6$.
    • If $x=0$, $y=6 \implies (0,6)$.
    • If $y=0$, $2x=6 \implies x=3 \implies (3,0)$.
    • Draw a solid line through $(0,6)$ and $(3,0)$ because the symbol is $\le$.
  2. Test Point: $(0,0)$.
  3. Substitute: $2(0) + (0) = 0 \le 6$. This statement is True.
  4. Shade: Shade the side of the line containing $(0,0)$, which is the region below and to the left of the line.
  • Common Mistake: Always remember the rule for solid vs. dashed lines based on whether the "equal to" part is included in the inequality symbol.

Systems of Linear Inequalities and Feasible Regions

In real-world scenarios, you often have multiple limitations, each represented by an inequality.

  • System of Linear Inequalities: Two or more linear inequalities considered together.
  • Feasible Region: The solution to a system of inequalities. It is the area on the coordinate plane where all shaded regions from each individual inequality overlap. Every point in the feasible region satisfies all the constraints simultaneously.

Non-Negativity Constraints ($x \ge 0, y \ge 0$) In business, quantities like production units, hours worked, or money spent cannot be negative. Therefore, constraints $x \ge 0$ and $y \ge 0$ are almost always included in linear programming problems. Graphically, these restrict the feasible region to the first quadrant of the coordinate plane (where both $x$ and $y$ values are positive or zero).

Example: Graph the feasible region for the system: $x + y \le 8$ $2x + y \le 12$ $x \ge 0$ $y \ge 0$

  1. Graph $x + y = 8$: Solid line through $(8,0)$ and $(0,8)$. Test $(0,0)$: $0+0 \le 8$ (True). Shade below/left.
  2. Graph $2x + y = 12$: Solid line through $(6,0)$ and $(0,12)$. Test $(0,0)$: $0+0 \le 12$ (True). Shade below/left.
  3. Graph $x \ge 0$: Shade the region to the right of the y-axis.
  4. Graph $y \ge 0$: Shade the region above the x-axis. The feasible region is the quadrilateral formed by the overlap of all these shaded areas.

Corner Points of the Feasible Region

Corner Points (Vertices): These are the points where the boundary lines of the feasible region intersect. They are extremely important because, as we will see, the optimal solution for a linear programming problem always occurs at one of these points.

How to Find Corner Points:

  1. Identify Intersecting Boundary Lines: Look at your graph of the feasible region to see which pairs of boundary lines form the corners.
  2. Solve Simultaneously: For each pair of intersecting lines, solve their equations as a system (e.g., using substitution or elimination) to find the exact coordinates of the intersection point.
  3. Verify: Check that each point you find actually lies on the edge of the feasible region (i.e., it satisfies all the original inequalities).

Example: Find the corner points for the previous system: $x + y = 8$ $2x + y = 12$ $x = 0$ (y-axis) $y = 0$ (x-axis)

  • Corner 1: Intersection of $x = 0$ and $y = 0 \Rightarrow \mathbf{(0, 0)}$
  • Corner 2: Intersection of $2x + y = 12$ and $y = 0$
    • $2x + 0 = 12 \Rightarrow x = 6 \Rightarrow \mathbf{(6, 0)}$
  • Corner 3: Intersection of $x + y = 8$ and $2x + y = 12$
    • Subtract $(x+y=8)$ from $(2x+y=12)$: $(2x+y)-(x+y) = 12-8 \Rightarrow x = 4$.
    • Substitute $x=4$ into $x+y=8$: $4+y=8 \Rightarrow y=4 \Rightarrow \mathbf{(4, 4)}$
  • Corner 4: Intersection of $x + y = 8$ and $x = 0$
    • $0 + y = 8 \Rightarrow y = 8 \Rightarrow \mathbf{(0, 8)}$ The corner points are $(0, 0)$, $(6, 0)$, $(4, 4)$, and $(0, 8)$.

The Objective Function and Optimization

Once the feasible region and its corner points are found, the next step is to use them to optimize an outcome.

  • Objective Function: This is the mathematical expression ($Z = ax + by$) that you want to maximize (e.g., profit, revenue) or minimize (e.g., cost, waste). $x$ and $y$ are the decision variables that you are trying to find optimal values for.
  • Linear Programming: The overall process of finding the maximum or minimum value of an objective function subject to a set of linear constraints.

The Corner Point Theorem This theorem is the cornerstone of linear programming. It states:

  • If a linear programming problem has an optimal solution (maximum or minimum), that solution will always occur at one of the corner points of the feasible region.
    • This means you don't need to test every point in the infinite feasible region; you only need to evaluate the objective function at a finite number of corner points.

Steps to Find the Optimal Solution:

  1. Find all corner points of the feasible region.
  2. Evaluate the objective function ($Z = ax + by$) at each corner point.
  3. The largest value of $Z$ is the maximum; the smallest value of $Z$ is the minimum.

Example: Maximize $Z = 5x + 4y$ subject to the constraints (from the previous example): Corner points: $(0, 0)$, $(6, 0)$, $(4, 4)$, $(0, 8)$.

  • At $(0, 0)$: $Z = 5(0) + 4(0) = 0$
  • At $(6, 0)$: $Z = 5(6) + 4(0) = 30$
  • At $(4, 4)$: $Z = 5(4) + 4(4) = 20 + 16 = 36$
  • At $(0, 8)$: $Z = 5(0) + 4(8) = 32$

The maximum value of $Z$ is 36, occurring at the point (4, 4).

Example: Minimize $Z = 3x + 2y$ subject to: $x + y \ge 6$ $2x + y \ge 8$ $x \ge 0$ $y \ge 0$

  1. Graph Feasible Region:

    • $x+y=6$: Solid line through $(6,0)$ and $(0,6)$. Test $(0,0)$: $0+0 \ge 6$ (False). Shade above/right.
    • $2x+y=8$: Solid line through $(4,0)$ and $(0,8)$. Test $(0,0)$: $0+0 \ge 8$ (False). Shade above/right.
    • $x \ge 0$, $y \ge 0$: First quadrant. The feasible region is unbounded, extending upward and to the right.

  2. Find Corner Points:

    • Intersection of $x=0$ and $x+y=6$: $(0,6)$. (Check $2x+y \ge 8$: $0+6 \not\ge 8$. This point is not in the feasible region.)
    • Intersection of $y=0$ and $2x+y=8$: $(4,0)$. (Check $x+y \ge 6$: $4+0 \not\ge 6$. This point is not in the feasible region.)
    • Intersection of $x+y=6$ and $2x+y=8$:
      • Subtract: $(2x+y)-(x+y) = 8-6 \Rightarrow x=2$.
      • Substitute $x=2$ into $x+y=6$: $2+y=6 \Rightarrow y=4$. Point: $\mathbf{(2,4)}$.
    • Other corner points of the actual feasible region are the y-intercept of $2x+y=8$ if it's within the region and the x-intercept of $x+y=6$ if it's within the region. However, a careful graph shows the actual boundary lines forming corners for this unbounded region are $x=0$, $y=0$, and the two lines. The corners are formed by:
      • $x=0$ and $2x+y=8 \implies (0,8)$.
      • $y=0$ and $x+y=6 \implies (6,0)$.
      • And the intersection of $x+y=6$ and $2x+y=8 \implies (2,4)$.
      • Wait, re-read the example's corner point analysis: The example points out that $(0,6)$ and $(4,0)$ are not in the feasible region. This is crucial for unbounded regions. The correct corner points are $(0,8)$, $(2,4)$, and $(6,0)$.

  3. Evaluate $Z = 3x + 2y$:

    • At $(0, 8)$: $Z = 3(0) + 2(8) = 16$
    • At $(2, 4)$: $Z = 3(2) + 2(4) = 6 + 8 = 14$
    • At $(6, 0)$: $Z = 3(6) + 2(0) = 18$ The minimum value of $Z$ is 14, occurring at the point (2, 4). (For unbounded regions, a minimum or maximum may not exist if the region extends infinitely in the "favorable" direction, but here a minimum clearly does).

Summary of the Linear Programming Process

  1. Define Variables: Clearly identify what $x$ and $y$ represent (e.g., number of units, hours).
  2. Formulate Objective Function: Write the equation $Z = ax + by$ that you want to optimize.
  3. Formulate Constraints: Translate problem conditions into linear inequalities, including $x \ge 0, y \ge 0$.
  4. Graph Feasible Region: Plot all boundary lines and shade the region that satisfies all inequalities.
  5. Find Corner Points: Identify and calculate the coordinates of all vertices of the feasible region.
  6. Evaluate Objective Function: Substitute the coordinates of each corner point into the objective function to find the corresponding $Z$ value.
  7. Identify Optimal Solution: Choose the corner point that yields the maximum or minimum $Z$ value, as required by the problem, and state it in context.
