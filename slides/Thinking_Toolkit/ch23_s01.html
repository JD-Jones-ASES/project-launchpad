Chapter 23, Section 1

Gall's Law

Study Slides

Key Concepts

Gall's Law

— Working complex systems always evolve from working simple systems; complex systems designed from scratch consistently fail

Emergent behavior

— Properties that arise from the interaction of system components and cannot be predicted from the components alone; the reason designed complexity fails

Incremental development

— The reliable path to working complexity: build something simple that works, test it, then add one layer at a time

Waterfall model

— The failing design approach: specify everything upfront, build the complete system, then test — produces spectacular failures with remarkable regularity

Important Terms

John Gall

American pediatrician whose 1975 book Systemantics drew on decades of observations about how systems actually behave (as opposed to how designers expect them to)

Working prototype

A simple, functional version of a system that serves as the foundation for incremental complexity — the required starting point, per Gall's Law

Emergent failure

System failure that results not from any single defective component but from unpredicted interactions between components that each function correctly in isolation

Hubris in design

The temptation to design a perfect, comprehensive system from the ground up — what Gall's Law identifies as nearly always fatal to the project
