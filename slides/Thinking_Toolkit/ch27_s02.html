The Halting Problem — Study Guide

Study Slides

Key Concepts (1/2)

The Halting Problem

The question of whether a given program with a given input will eventually stop running or loop forever — proven undecidable by Alan Turing in 1936

Undecidability

A problem is undecidable if no algorithm can correctly answer it for all possible inputs — not merely difficult but provably impossible

Proof by Contradiction with Self-Reference

Assume a halting detector exists, construct a program that does the opposite of what the detector predicts about itself, derive a contradiction

Permanent Computational Boundary

The impossibility is not a limitation of current hardware or software — no conceivable computer can ever solve the Halting Problem in general

Computational Analogue of Incompleteness

The Halting Problem is to computer science what Godel's theorems are to mathematics — a fundamental limit on what any formal system can decide

Key Concepts (2/2)

Practical Implications

Antivirus, compiler optimization, and program verification are all bounded by the undecidability of the Halting Problem

Important Terms (1/2)

Alan Turing

British mathematician (1912–1954), pioneer of computer science; proved the Halting Problem undecidable at age 23 in "On Computable Numbers, with an Application to the Entscheidungsproblem" (1936); ...

Entscheidungsproblem

German for "decision problem" — Hilbert's challenge to find an algorithm deciding the truth or falsity of any mathematical statement; Turing (and independently Church) proved this impossible

Turing machine

Turing's theoretical model of a general-purpose computer — a simple abstract device that can simulate any algorithm; the theoretical foundation of computer science

Halting detector H

The hypothetical program whose existence Turing's proof assumes (for contradiction); given any program P and input I, H(P, I) correctly predicts whether P halts on I

Diabolical program D

Constructed from H; takes a program P as input, asks H whether P halts on itself, then does the opposite of what H predicts

Important Terms (2/2)

Undecidable problem

A problem for which no algorithm can produce a correct answer for all possible inputs; the Halting Problem is the canonical example
