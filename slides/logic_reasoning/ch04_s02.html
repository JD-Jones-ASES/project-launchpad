Logical Connectives

The basic operations used to combine or modify propositions.

Order of Operations for Logic

A rule set for evaluating compound statements, similar to mathematical order of operations: Parentheses first, then NOT, then AND, then OR, then IF THEN.

Compound Statements

Logical statements formed by combining two or more simple propositions using connectives, which require step-by-step evaluation using truth tables.

Tautology

A statement that is always true, regardless of the truth values of its individual propositions. Modus Ponens, when written as a single formula, is an example of a tautology.

Important Terms (1/2)

Truth Table

A tabular display showing all possible truth values for a simple or compound logical statement, given every possible combination of truth values for its constituent simple propositions.

Proposition

A declarative sentence that is either true or false, but not both.

Connective

A logical operator (like AND, OR, NOT, IF THEN) that joins or modifies propositions to form more complex statements.

Conditional Statement ($P \to Q$)

An "if-then" statement where $P$ is the antecedent (the "if" part) and $Q$ is the consequent (the "then" part).

Tautology

A compound statement that is always true, no matter what the truth values of its simple component propositions are.

Important Terms (2/2)

Converse

For a conditional $P \to Q$, its converse is $Q \to P$.

Inverse

For a conditional $P \to Q$, its inverse is $\sim P \to \sim Q$.

Contrapositive

For a conditional $P \to Q$, its contrapositive is $\sim Q \to \sim P$.

Logically Equivalent

Two statements are logically equivalent if they always have the same truth value in every possible scenario (i.e., their truth tables are identical).

Boolean Algebra

A system of algebra developed by George Boole that uses symbols for logical operations (AND, OR, NOT) to represent logical statements mathematically, forming the basis for computer logic.

The Converse, Inverse, and Contrapositive

Original Conditional

$P \to Q$ ("If it rains, then the ground is wet.")

Converse

$Q \to P$ ("If the ground is wet, then it rains.")

Inverse

$\sim P \to \sim Q$ ("If it doesn't rain, then the ground isn't wet.")

Contrapositive

$\sim Q \to \sim P$ ("If the ground isn't wet, then it didn't rain.")

How does the 'NOT' connective (¬P) affect the truth value of a proposition P?

The 'NOT' connective simply flips the truth value of the proposition. If P is true, ¬P is false; if P is false, ¬P is true.

An 'AND' statement (P ∧ Q) is true only when both propositions P and Q are true. If even one part is false, the entire statement is false.

Key Concepts (2/2)

What is the correct order of operations for evaluating compound logical statements?

The order is: 1. Parentheses (innermost first) 2. NOT (¬) 3. AND (∧) 4. OR (∨) 5. IF THEN (→).

When building a truth table for a compound statement like ¬P ∧ Q, what is the first step after listing P and Q?

The first step is to create a helper column for ¬P by flipping the truth values of P, and then combine that result with Q using the AND connective.

What is a tautology?

A tautology is a statement that is always true, regardless of the truth values of its constituent propositions. Its truth table column will be entirely 'True'.

Which logical tautology is represented by the formula [P ∧ (P → Q)] → Q?

This formula represents Modus Ponens, which is a fundamental rule of inference stating that if P is true and 'if P then Q' is true, then Q must be true.

Given a conditional statement P → Q, what is its converse?

The converse of P → Q is Q → P (the antecedent and consequent are swapped).

Given a conditional statement P → Q, what is its inverse?

The inverse of P → Q is ¬P → ¬Q (both the antecedent and consequent are negated).
