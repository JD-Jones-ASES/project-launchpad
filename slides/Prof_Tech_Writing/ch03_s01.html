Clear and Effective: Professional &amp; Technical Writing

Chapter 3, Section 1

Writing Numbered Steps in Instructions

Study Slides

Key Concepts (1/2)

Imperative Mood

Every step should be a direct command starting with a verb, telling the reader exactly what to do. This ensures clarity, conciseness, and an action-oriented tone.

One Action Per Step

Each numbered step should contain only one discrete action. Combining multiple actions can confuse readers and lead to skipped parts of a procedure.

Substeps

Use lettered or indented substeps for smaller actions that logically group under a single main step, enhancing organization without breaking the flow.

Numbered vs. Bulleted Lists

Use numbered lists when the order of steps is crucial (most instructions). Use bulleted lists only when items can be completed in any order (e.g., a list of materials).

Consistent Formatting

Maintain uniform formatting for text elements (e.g., bolding button names, italicizing menu options) throughout the instructions to reduce reader confusion.

Key Concepts (2/2)

Expected Results

Include the anticipated outcome after critical steps to help readers confirm they have successfully completed the action.

Important Terms

Imperative Mood

A grammatical mood used to express direct commands or requests. In instructions, steps written in this mood start with a verb and address the reader directly.

Substep

A smaller action or mini-step nested within a main numbered step, typically indicated by letters (a), (b), etc., or indentation.

Sequential Steps

Actions arranged in a specific order where the completion of one step is dependent on the completion of the previous one.

Parallel Steps

Actions or tasks that can be performed simultaneously, in any order, or independently of each other.

Run-On Step

A common mistake where multiple unrelated or distinct actions are combined into a single numbered step, causing confusion for the reader.

Using the Imperative Mood (1/2)

Beginning with a verb

Each step starts directly with an action verb (e.g., "Remove," "Press," "Click").

Direct address

It implicitly speaks directly to the reader ("You").

Clarity and Conciseness

It avoids unnecessary words and ambiguity, focusing solely on the action required.

Correct (Imperative)

"Remove the back panel."

Avoid (Passive)

"The back panel should be removed." (Hides who performs the action)

Using the Imperative Mood (2/2)

Avoid (Wordy)

"You will want to go ahead and remove the back panel at this point." (Buries the action in filler)

Avoid (Conditional)

"The user may remove the back panel if desired." (Makes the step seem optional)

One Action Per Step

Why it matters

When multiple actions are packed into one step, readers can lose track, complete only part of the step, or become confused about whether they've finished.

Common Mistake: The Run-On Step

Do not combine steps to "save space." Readers follow instructions sequentially, completing one step before moving to the next. Combining "Remove the old filter" and "Rinse the housing" into a singl...

Instead of (Multiple Actions)

"Open the lid, remove the old filter, rinse the filter housing with warm water, dry it thoroughly, insert the new filter, and close the lid."

Use (One Action Per Step)

When to Use Substeps

Purpose

Substeps (e.g., (a), (b), (c)) are used when smaller actions logically belong together under a single umbrella task, and numbering them as separate main steps would break the context.

Formatting

Substeps are typically lettered or indented beneath the main step.

Depth Limit

Avoid nesting substeps more than one level deep. If you find yourself needing sub-sub-steps, it's an indication that the instructions should be broken into separate main sections or procedures.

Numbering and Formatting Guidelines (1/2)

Numbered Lists

Use for sequential steps where the order is critical. This is the default for most instructions.

Bulleted Lists

Use only for items where the order does not matter (e.g., a list of materials needed, or parallel tasks that can be done in any sequence).

Start with a Verb

Every step should begin with an action verb (e.g., "Click," "Open," "Remove," "Insert").

Keep Steps Short

Aim for one to two sentences per step. Lengthy explanations should be placed before the step or in a dedicated "Note" section.

Use Consistent Formatting

Apply the same formatting (e.g., bolding, italics) to similar elements (e.g., button names, menu options) throughout the entire set of instructions. Inconsistency can lead to confusion.

Numbering and Formatting Guidelines (2/2)

Include Expected Results

After critical steps, describe what the reader should see or experience. This confirms they have completed the step correctly (e.g., "The status light turns green," "A confirmation dialog appears").

Model Document: Setting Up a New Employee Email Account (1/2)

Clear Title, Purpose, Prerequisites

Sets the stage for the user.

Numbered Steps

Used for the main procedure, indicating sequential actions.

Imperative Mood

Each step starts with a verb (e.g., "Open," "Enter," "Click," "Create").

Substeps

Used for the password requirements (a, b, c) under the main "Create a new password" step, as these are related sub-actions.

Expected Results

Included after key actions (e.g., "The system displays a 'Change Your Password' screen," "A confirmation message appears: 'Password updated successfully.'").

Result Statement and Troubleshooting
