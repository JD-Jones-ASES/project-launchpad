Important Terms (1/2)

Task-Oriented Writing

A documentation approach that emphasizes user actions and goals, typically by starting headings and instructions with verbs.

Feature-Oriented Writing

A documentation approach that describes the components or features of a software, rather than what a user can accomplish with them.

Consistent Terminology

The practice of using the exact same word or phrase every time a particular concept or element is referenced in documentation.

Getting Started Section

The introductory part of a user guide designed to help new users quickly set up and perform a basic function with the software.

Screenshot

A digital image of what is visible on a computer screen, used in documentation to visually illustrate software interfaces or results.

Important Terms (2/2)

Callout

An annotation (such as a numbered circle, arrow, or highlighted box) added to a screenshot to direct the user's attention to a specific area or element.

Tooltip

A small, brief piece of explanatory text that appears when a user hovers over or taps a graphical user interface element in a software application.

Quick-Reference Card

A concise, often single-page document that provides essential, frequently accessed information and instructions in an easy-to-digest format.

Task-Oriented Writing

Core Principle

Every part of your documentation—headings, steps, topic sentences—should begin with a verb and describe an action the user wants to take.

Consistent Terminology

The Rule

Pick one specific word for each concept or interface element and use it uniformly throughout your entire document.

Why it Matters

Inconsistent terminology forces readers to constantly wonder if different words refer to the same thing or different things, slowing them down and causing confusion.
